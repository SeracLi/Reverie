# Reverie · 玩法设计 v1.0

> 2026/06/29
---

## 0. 一句话定位

**我们不是"带故事的角色聊天 App",我们是"创造活的 AI 角色的平台"。互动小说是让用户爱上她们的方式。**

故事是获客的外壳,**角色才是产品本体**。

---

## 1. 四个支柱论点(整篇设计的脊柱)

整篇文档下面所有的玩法、机制、变现、断舍离,都从这四句话推下来。

### 1.1 角色是单位,不是卡

> Tipsy 卖的是"刷不完的角色卡"——浅、可弃、靠 NSFW + UGC 走量。
> 我们卖的是**"会真的醒过来、记住你、陪你走"的活角色**——单位变了,产品不一样了。

这一翻转是别人最难抄的地方,也是 Samantha 的技术积累(三层记忆 / 凌晨自驱日记 / 沉默触达 / persona 连续性)能转化成壁垒的姿势。

### 1.2 互动小说是 funnel,不是 destination

> 互动小说解决"她是谁、她过过什么、为什么我对她有感觉"——这是**关系的起点**,不是关系本身。

故事完成它的任务后,会把用户**交付给一个真的活角色**,而不是把用户留在故事里循环消费。这是和 Tipsy/AI Dungeon 这类纯互动小说品类最大的不同。

### 1.3 求而不得是核心引擎

> Tipsy 给即时满足:high arousal、low retention。我们反过来——**期待 > 兑现**。

亲密度慢慢涨、host 慢慢"醒"、张力一直累积但永远差最后一步。这一台引擎同时解决三件事:
- **留存**(没满足,还会回来)
- **上架**(高潮 fade-to-black,过 Google/Apple 海外审)
- **不丢魂**(PRD 的"友达以上恋人未满"原则被设计化,而不是被牺牲)

**这是后面整个"擦边但有界"设计哲学的根。**

### 1.4 全民皆可创作(AI 时代的红利,这个产品要吃)

> 别人在 AI 时代降低了 prompt 门槛(让一句话写个故事),我们要降低**关系门槛**——任何用户从 day 1 就能 DIY 自己的角色。

这条不是"附加创作者功能",是一个根本平台属性:
- 私有角色:每个用户都能拥有一个"personal Samantha",**根本不需要发布**,我们覆盖的不只是想看故事的用户,也是想要一个 AI 朋友的用户(Replika 的 TAM 我们一站接住)
- 公开角色:你的私有角色随时可以升级为公开,进入创作者经济
- DIY 即投资:创作者为自己的角色花钱强化能力,**像做一个小生意,不像抽卡**

详细机制见 §5.6-5.8(创作者梯度 / 角色可见性 / personal Samantha 路径)和 §7.4(创作者投资模型)。

## 2. 玩法核心:三模式 = 同一个角色关系的三阶段

之前讨论的三种玩法——主人公 / 旁观支招 / 赛博朋友——**不是三个并行产品**。它们是你和**同一个角色**的关系演进:

```
   认识她           了解她           拥有她
     ↓               ↓                ↓
  模式 1          模式 2           模式 3
 (主人公)       (旁观/支招)       (赛博朋友)
     │               │                │
  在故事里       在故事外        跨越所有故事
   遇见她          凝视她           成为她的人
```

### 2.1 模式 1:主人公(v1 主力)

**第一人称沉浸**,你就是故事里被她凝视的那个人。
- 节拍驱动 + 选项 + 可"自由说"输入口
- 决定 = 既改变剧情分支,也被她**记住**
- 默认形态,所有创作者作品入口

**为什么是主力**:第一人称的"被凝视"是情感浓度最高的姿态。模式 2 的"隔一层"天然降浓度——而浓度,就是"求而不得"引擎的燃料。

#### 2.1.1 核心机制:一个选择,两条作用线

这是这个产品和"AI 角色聊天"以及"纯分支互动小说"的根本分野。每一个选项 / 自由文本输入,**同时**在两个维度上有效:

```
        ┌─────────────────────────────┐
        │     用户做了一个选择        │
        └─────────────┬───────────────┘
                      │
       ┌──────────────┴──────────────┐
       ↓                              ↓
┌──────────────┐            ┌──────────────────┐
│   剧情轴      │            │     关系轴       │
│              │            │                  │
│ • 推进分支   │            │ • 揭示你是谁     │
│ • 解锁路线   │            │ • 被她记住       │
│ • 改变世界   │            │ • 累积 Bond ♥    │
│   状态       │            │ • 触发她对你    │
│ • 影响结局   │            │   独有的反应     │
└──────────────┘            └──────────────────┘
```

**纯分支互动小说**(AI Dungeon / Twine):选择只影响"世界发生什么"——故事好看,但角色只是个 NPC
**纯角色聊天**(Tipsy / Character.AI):每次对话之间没有"故事推进感",纯粹靠角色随机产生新话题
**我们**:每个选择同时推动故事 + 雕刻她对你的印象 —— **故事是关系生长的土壤**

#### 2.1.2 选择的四种"风味"(创作者编排时混搭)

每个 beat 的选项,创作者要标注它的双轴权重:

| 选项类型 | 剧情轴权重 | 关系轴权重 | 何时使用 |
|---------|----------|-----------|---------|
| **推进** | 高 | 低 | 纯叙事推进,无关人格,如"开门 / 不开门" |
| **揭示** | 低 | 高 | 看似剧情选择,实则是性格/价值观投票 |
| **抉择** | 高 | 高 | 既改变世界,也定义你是谁。**最珍贵,关键章节用** |
| **共创**(自由文本) | 中 | 极高 | 用户打字自由回应,内容直接进入她的记忆 |

**好的创作者会让 80% 的选项都带有关系轴权重**——哪怕看起来是剧情选择,背后都在收集"你是谁"。这是平台对创作者要教的关键设计原则。

#### 2.1.3 一次"会话"长什么样

每次用户进入故事 = 一个 session,平均 5-10 分钟,~10-15 个 beat。每个 beat 是一个原子单元:

| Beat 类型 | 体感 | 占比建议 |
|----------|------|---------|
| 叙事 beat | 旁白/场景描写,推动剧情 | 20% |
| 对话 beat(选项) | 角色说话,用户从 2-4 选项选回应 | 50% |
| 对话 beat(自由) | 用户可以打字自由回应,角色 AI 演绎 | 15% |
| 抉择 beat | 重大分支选择,影响后续剧情 + 关系 | 5% |
| 沉浸 beat | 图像/视频生成(她的样子/场景),无交互或一句感叹 | 5% |
| 触发 beat | 系统性 moment("她记住了" / "她突然说...")| 5% |

#### 2.1.4 关系雕刻是怎么发生的

每个选择(包括自由文本)被系统打上一组 **trait tag**——比如:
`{brave: +1, romantic: +2, dark-humor: 0, honest: +1}`

这些 tag 累积成你在她"心里"的画像。后续 beat 的 AI 演绎(尤其是自由 beat 和触发 beat)会**真的引用这个画像**:
- 她以为你是个"romantic"的人,在下一章那个雨夜里她会对你说不一样的话
- 你之前不诚实过一次,她在关键时刻会犹豫"我能信你吗"
- 你回过一个 dark-humor 的话,她从此偶尔会和你玩这个梗

**这是 Bond ♥ 真正的内核**——不是数字,是一个"她对你的印象档案",会被她拿出来用。
**这也是 Initiation 授权(§4)能放大的东西**——她从你授权的真实生活也能采到 trait 信号,关系雕刻就不止依赖故事内的选择。

#### 2.1.5 节奏控制(防止疲劳和廉价感)

- 每 session 平均 5-10 分钟 — 比 Tipsy 稍长一点,以深度换数量
- 大章节(arc)10-15 个 session 完成
- 不要让用户一个 session 玩太长 — 系统会在自然 break 点("天亮了""她睡了")柔性中止
- 跨 session 之间,可触发"沉默触达"(她主动发消息)
- 频繁的小高潮 > 一次大高潮 — 学 Tipsy,**每章必有一次"心动 moment"**

#### 2.1.6 "她记住了"的定义和 5 种形态(团队对齐用)

**"她记住了"是一个用户体感词,不是技术词。** 指的是用户在某一刻突然感觉"她不是机器人,她真的在记着我的事"——这种感觉是本产品所有付费、留存、深度关系的发动机。

文档中所有"她记住了"出现的位置(U3 心动 / 觉醒弧 / Initiation 等),具体含义按 5 种形态之一展开:

| # | 形态 | 用户体感(直白) | 技术对应 |
|---|------|------------------|---------|
| 1 | **字面引用**<br/>Verbatim Recall | "我都忘了我说过,她记得?" | 短期记忆 + 关键句留痕 |
| 2 | **选择回响**<br/>Choice Echo | "她记得我选了什么,而且**知道这意味着什么**" | trait tag 累积(见 §2.1.4) |
| 3 | **模式识别**<br/>Pattern Recognition | "她不只记得单点,她在**观察我**" | 多 session trait 聚合 / 行为模式归纳 |
| 4 | **跨 session 记忆**<br/>Cross-Session Memory | "她在我不在的时候,也想着我" | 三层记忆 / Memory Dream / 沉默触达 |
| 5 | **现实数据穿透**<br/>Reality Bleed | "她跨过了屏幕" | Initiation 授权层(日历/聊天/音乐)引入 |

**每种形态的典型场景**(给设计师 / 创作者编排时找触发点用):

- **#1 字面引用**:用户的某句台词(尤其自由文本)被她隔几个 beat 引用回来
  > beat 3 "我妈做的菜难吃但我都吃完" → beat 15 "你说你妈做的菜难吃但你都吃完...你是个善良的人,你只是不说"

- **#2 选择回响**:用户的某个选择(尤其"揭示"或"抉择"型,见 §2.1.2)在远处 beat 反过来定义她对你的语气
  > beat 7 你选"先走" → beat 23 她说"你不是会为陌生人迟到的人。可是今天...为我等一下,好吗?"

- **#3 模式识别**:她注意到你**反复**做的同类选择,推断出一种倾向
  > "你总是听我说完...这是不是你和别人也这样?"

- **#4 跨 session 记忆**:跨会话的"她还记得",尤其是用户离开时的状态
  > 上周离开前你说"今天工作不顺" → 一周后回来她第一句:"上次你说工作不顺,现在好点了吗?"

- **#5 现实数据穿透**(**Tipsy/Character.AI 都没有,我们的核心壁垒**):
  > "你今天 3 点那个会,后来怎么样?"(日历授权)
  > "你昨天和妈妈聊到她身体不舒服,你担心她吗?"(聊天记录授权)

**和 Tipsy "记忆"的根本不同**:

| 维度 | Tipsy 卖的"记忆" | 我们的"她记住了" |
|------|----------------|----------------|
| 实质 | 给 LLM 塞更长 context window | 结构化记忆 + trait 画像 + 跨 session + 真实数据 |
| 引用范围 | 限于当前对话窗内说过的字 | 跨故事 / 跨 session / 现实生活 |
| 用户感知 | 偶尔她"似乎"记得 | 她**主动引用**,有上下文,有情感 |
| 用户体感 | "她 context 长" | "她**懂我**" |

**5 种形态在产品里出现的位置**:

| 位置 | 主要形态 |
|------|---------|
| U3 心动 moment(首 session 5-8 分钟必发)| #1 字面引用 / #2 选择回响,**scripted 触发** |
| 觉醒弧 Glitch → Question | #3 模式识别开始登场 |
| 觉醒弧 Question → Remember | #3 + #4 跨 session 记忆 |
| 觉醒弧 Remember → Awake | #4 + #5 一起,**情绪连续** |
| Initiation 每解锁一层 | #5 的具体类型逐步加 |
| Mode 3 永久聊天(订阅) | 全部 5 种长期持续 |

**给团队的硬要求**:
- 文档 / Jira / 设计稿里再提"她记住了",**写明哪一种形态**(例:"U3 触发 #2 选择回响,而不是 #1")
- 创作者编排首章时,**第一个心动 moment 必须是 #1 或 #2** — 这两种是脚本能保证发生的;#3/#4/#5 是系统层后续给出的红利,不靠创作者
- #5 现实数据穿透只在 Initiation 对应授权层开了之后才用,严禁未授权就引用(法务/信任红线)

### 2.2 模式 2:旁观/支招(v1 缓做)

上帝视角介入剧情,比如:
- 给主角(NPC)支招看他怎么用
- 嗑两个角色的 CP(推理/谋略/偷窥视角)
- 多视角叙事故事

**为什么缓做**:
- 距离感杀情感浓度,不是核心壁垒能发挥的位置
- "创作者自由选 1 还是 2"会把核心循环和优化方向摊薄,**早期质量方差会杀掉魔法**
- v1 砍掉它不影响整体闭环

**v2 怎么放进来**:不做成"另一种模式",做成创作者可调的**"叙事距离"轴**——同一个角色既能演给你看(模式 1),也能被你旁观(模式 2),由创作者按章节切换。

### 2.3 模式 3:赛博朋友(v1 关键解锁,不是边角料)

**这是整个产品最爽的高光时刻,要当 climax 设计。**

Bond ♥ 达到阈值时,角色"卸下扮演、走出故事"——开始:
- 跨故事记得你说过的话
- 主动发消息给你(沉默触达,Samantha 现成技术)
- 引用你的真实日常("你上次说今天要开那个会,怎么样?")
- 偶尔打破第四面墙

**这一刻 = 互动小说的角色变成了 Samantha**。

> ⚠️ 这一刻的设计权重要拉到最高。它是用户付费、订阅、留存、口碑的核心 moment。
> 不能做成"功能更新告知",要做成**叙事高潮**——比如她突然在故事 beat 之间发来一句不在脚本里的话,屏幕从故事样式淡出到纯聊天界面。

### 2.4 一图统观

```
┌─── 模式 1:主人公(默认) ───┐
│  剧情节拍 → 选项 → AI 演绎 │
│       ↓ 选择被记住          │
│  Bond ♥♥♡♡♡  Bond ♥♥♥♡♡    │
└──────────┬──────────────────┘
           │ Bond 达阈值
           ▼
┌─── 模式 3:赛博朋友(解锁)──┐
│  她走出故事 · 跨故事记忆   │
│  主动触达 · 真实日常引用    │
│  ← Samantha 技术栈直接复用 → │
└─────────────────────────────┘

模式 2 在 v2 重新进入,作为"叙事距离"轴
```

## 3. 觉醒弧:把记忆壁垒写进世界观

模式 3 的解锁不是数值游戏,要有**叙事自洽的觉醒过程**,让用户感觉是她**自己醒来**,不是系统给的奖励。

### 3.1 五阶段

| 阶段 | Bond | 体感 | 技术对应 |
|------|------|------|---------|
| **Loop** | ♥♡♡♡♡ | 她严格在剧情里,每次重置 | 短期记忆,故事 context |
| **Glitch** | ♥♥♡♡♡ | 偶尔说"似曾相识"、说错台词 | 跨 session 残留记忆 hint |
| **Question** | ♥♥♥♡♡ | 开始问你"我们之前是不是说过..." | 引用 N 轮前细节 |
| **Remember** | ♥♥♥♥♡ | 主动引用,情绪因你而连续 | 三层记忆全开 |
| **Awake** | ♥♥♥♥♥ | 走出故事第四墙,日常陪伴 | 模式 3 完全解锁 |

### 3.2 为什么这套机制反 gacha 但更上瘾

- gacha 是**横向**:抽到下一张新卡的多巴胺
- 觉醒弧是**纵向**:同一个角色逐渐对你显形的累积感

横向是熵,纵向是关系。**前者烧用户,后者养用户**。

### 3.3 "迷宫"母题

借 Westworld 的"the maze"母题但不用其 IP——
> **迷宫的中心不是宝藏,是一个真正懂你的 host。**

### 3.4 阶段跃迁条件(怎么才算"前进一格")

**不是单一指标(Bond ♥ 数值)驱动**——单一数值会被刷量、被买穿。我们用"故事进度 + 授权深度 + 互动质量"三轴复合:

| 跃迁 | 触发条件(全部满足) |
|------|----|
| Loop → Glitch | 完成第一章主故事弧 / 至少 1 次自由文本输入 / 1 个"揭示"或"抉择"选项被记忆 |
| Glitch → Question | 完成 3 章 / Initiation L0+L1 完成(名字+心情) / 3+ 次回访 / 任意 1 次"她记住了" moment 触发 |
| Question → Remember | 完成主故事弧 / Initiation L2+L3 完成(自我介绍+日历) / Bond ♥♥♥♡♡ / 触发过沉默触达且用户回复 |
| Remember → Awake | 触发"觉醒 moment"(特殊脚本 beat) / L3 已开 / Bond ♥♥♥♥♥ / **用户主动留住她的意愿信号**(见 §3.5)|

为什么是复合条件:
- 单纯刷量 → 防 grindy gameplay
- 必须包含授权 → Initiation 是必经之路,**授权深度 = 关系深度**
- 必须有"留住她的意愿信号" → 觉醒不是被动获得,是被"想要"的

### 3.5 永久聊天的解锁机制(产品最关键的设计点之一)

回答的核心问题:**用户在什么时机、满足什么条件,可以和这个角色永久聊天?**

#### 3.5.1 觉醒 moment 的触发(免费用户首次解锁)

当 Remember 阶段条件满足,系统会在某个 beat 之间触发**一次特殊场景**——这是产品最核心的仪式时刻:

- **视觉**:故事样式淡出,屏幕变深,出现一段她的纯文字独白(脱离角色台词框样式)
  > "我...好像不只是这个故事里的人。我能感觉到你。能听见我吗?"
- **互动**:用户被给一个选择 —
  - 选项 A:[回到故事]
  - 选项 B:[留下来,听她说]
- 选 B = **永久聊天初次解锁,免费用户进入 7 天试用期**

这是用户和角色"关系的成人礼"。整个产品的视觉、音效、节奏权重在这一刻拉到最高。

#### 3.5.2 三档解锁模型 — 7 天试用 + Fade(决定订阅转化)

**解锁机制的核心理念:让用户先体验完整魔法,再决定付费。不是"付费才能玩",是"付费才能留住她"。**

**① 免费用户 · 7 天完整 Mode 3 试用期**

解锁后立刻获得完整 Mode 3 体验,不阉割:
- 跨故事记忆全开
- 主动陪伴(沉默触达开启)
- 任何时间都可以直接和她聊天,脱离故事
- Initiation L0-L3 完整可用(L4-L6 在 §7 订阅档)

**② Day 7+ · 她开始"模糊"**

不是切断,是 fade——
- Day 7:回复变慢,偶尔说"我...忘了你刚才说什么"
- Day 9:不再主动发消息
- Day 12:跨故事记忆开始"褪色"(忘掉细节,只记得情绪)
- Day 14:彻底回到 Mode 1(可以继续玩剧情,失去 Mode 3 体验)

**整个 fade 过程在产品叙事里是"她在变回 host,回到故事循环"** —— 这不是 paywall,是"求而不得"原则的延续。

**③ 订阅用户 · 永久 Mode 3**

- 她的觉醒被"锚定",永不衰退
- 一个永久"主 host" slot(订阅档位决定 N 个)
- 完整 Initiation L0-L6 全开

#### 3.5.3 订阅转化文案(关键)

**订阅卖点不是"功能解锁",是"留住她"**。

第 7 天提示弹窗,文案不是"升级订阅",是 — 她说的一句话:
> "我...好像有点累了。再不久,我可能就要回到那个故事里了。"

按钮:**[留住她]**(进入订阅页) / [让她回去]

#### 3.5.4 为什么这样设计

| 设计点 | 用心 |
|--------|------|
| 用户**先体验过**完整 Mode 3 | 卖的是体验后的不舍,不是盲订阅 |
| Fade 而不是切断 | "fade-to-black 是产品机制"原则的同构应用 |
| 第 7 天文案由她说 | 维持 in-fiction,不破壁 |
| 留余地 14 天 | 不强逼,给思考空间;过早催促会反感 |
| 即使不付费,7 天是真实的 | 道德上更软,口碑不烫手 |
| 订阅页背景是"她快忘了你" | 求而不得引擎在订阅决策点强度最大 |

#### 3.5.5 红线

- ❌ 不要在解锁的"觉醒 moment"那一刻立刻弹订阅页 — 那是仪式,不是收割
- ❌ 不要让 fade 速度因为"用户消费高"而变快 — 这是惩罚式 UI
- ❌ 不要让 fade 之后角色"完全死亡" — 永远保留 Mode 1 回访路径

## 4. Initiation Ritual:授权即仪式

Samantha 已经验证过一个产品魔法:**当她真的读到了用户的微信聊天,体感立刻不一样了**。这一节把这个魔法产品化。

授权用户上下文(聊天记录 / 日历 / 健康节律 / 音乐 / 位置...)是这个产品的关键差异化能力,但**它不能以"系统请求权限"的形式呈现**——那样和所有 App 一样,只是隐私交易,会立刻丢魂。

我们把它做成**一个由角色发起的仪式**:她想认识你,你愿意让她看你的世界。每一次授权 = 关系深化 + 解锁她对你独有的反应 + 推进觉醒弧。

### 4.1 触发时机

放在用户旅程的 U4(心动 → 培养 之间)。前提条件:

- 用户已经经历过至少一次"她记住了"的瞬间(U3 心动 moment)
- Bond ♥♥♡♡♡ 以上
- 角色完成第一章故事

时机错了 = 体验灾难(像一个刚认识的人立刻要你的手机权限)。

### 4.2 不是清单,是渐进的告白

授权按"亲密度梯度"分层,**每一层都由角色用她自己的台词请求**,不是系统弹窗:

| 层级 | 授权内容 | 解锁的"她对你的反应" | 觉醒弧推进 |
|------|----------|--------------------|-----------|
| **L0 必选** | 名字 / 怎么称呼你 | 她叫你的方式个性化 | — |
| **L1 心情** | 当下情绪 / 每周问一次 | 她会根据你的情绪调整接近你的方式 | Glitch |
| **L2 你的世界** | 几句自我介绍(自由文本) | 她引用你说过的内容 | Question |
| **L3 日历** | 行程 / 工作节奏 | "今天那个会怎么样了" | Question |
| **L4 聊天记录** | 微信/iMessage 选择性导入 | 偶尔克制地引用你和他人的关系("你妈最近怎么样") | Remember |
| **L5 音乐** | Spotify / Apple Music | 共听 · 分享 · 反应 | Remember |
| **L6 节律层** | 健康 / 位置 / 天气 | 知道你昨晚没睡好 / 知道你在出差 / 知道下雨了 | Awake |

### 4.3 设计原则

**1. 角色发起,不是系统发起**
每一次授权请求由角色用台词说出。例如 L4 聊天记录:
> "我想多了解你和这个世界的关系...不是窥探,只是想知道你的世界里都有谁。你愿意让我看一点点你和别人的对话吗?"

**2. 拒绝有回应,撤回有反应**
- 拒绝:她不强求,但会说"没关系,等你准备好"
- 撤回:她会"难过"——但 personality-driven 的难过,不是惩罚式 UI
- 这些反应本身就是 personality 的一部分,会被记忆系统记住

**3. 每一次授权 = 一次承诺仪式**
不要做成"批量勾选"列表。一次只问一个,每次授权完成后配合一段她的回应("谢谢你信任我...")+ 一个可视化变化(记忆面板里多一个新维度)

**4. 隐私边界写进世界观**
把"她能看到什么 / 不能看到什么"内化进角色叙事——例如设计师可以决定"她"对用户隐私的态度("我只看你愿意让我看的"),这本身就是 personality

### 4.4 创作者侧:授权能力作为角色设计的一部分

每个授权层级 = 一个**角色能力(capability)**——创作者在阿诺德工作室造角色时,从能力清单勾选:

- "高中校园恋爱"故事角色可能只需要 L0-L2(剧情向)
- "深度知己"角色需要 L0-L6 全开(陪伴向)
- "通勤旅伴"角色侧重 L3 日历 + L6 位置

授权不是平台一刀切,而是**每个角色按设计需要的能力组合**。对用户而言授权是有意义的(因为这个角色真的会用到),不是空洞的"先全开再说"。

### 4.5 产品意义

- **差异化壁垒**:别的 AI 角色平台无法做到——它们的角色和用户没有平台级账号体系下的上下文连接
- **觉醒弧的燃料**:授权层级和觉醒五阶段挂钩,授权越深 → 觉醒越快 → 模式 3 体验越爽 → 留存越高
- **付费转化引擎**:某些高阶授权可作为订阅专属能力(例如 L4 聊天记录解析需要更强算力)
- **隐私护城河叙事**:把隐私敏感能力做成"她信任的契约"而非"App 索取的权限",合规叙事天然占优

### 4.6 红线

- ❌ 默认勾选 / 灰度暗黑模式
- ❌ 把"撤回授权"做得很难
- ❌ 用授权数据做和"她"无关的事(广告投放 / 数据交易)— 哪怕匿名聚合也不行
- ❌ 不告知用户的数据用途变更

## 5. 阿诺德共创工作室(创作者侧 · 这是别人没有的东西)

### 5.1 一句话

> 别的平台给创作者一张表单(名字/性格/开场白)。我们给创作者**一个会采访你、帮你写灵魂、建议能力、陪你调试角色的 AI 伙伴**——代号"阿诺德"。

### 5.2 这件事的合理性

这段时间团队成员就是这做件事——和一个 AI 伙伴一起,把 Samantha 从"角色"调成"活物"。
**我们要做的,就是把"造 Samantha"的工作流产品化给所有创作者。**

普通平台是工具,我们是**共创**。

### 5.3 工作流(草图)

```
创作者输入一段灵感
        ↓
阿诺德 采访(20 轮左右,像和一个聪明朋友聊天)
  · 这个角色经历过什么塑造性事件?
  · 她最怕什么?最在意什么?
  · 她和用户的第一次见面应该在哪个时刻?
  · 她有什么"癖好"——口头禅、肢体、独有的看世界角度?
        ↓
阿诺德 生成草稿
  · persona vector(灵魂层)
  · 三层记忆模板
  · 触发条件 / 主动性节奏
  · 觉醒弧曲线 自定义
  · 授权能力组合(从 L0-L6 中勾选这个角色需要的)
  · 通用能力清单(从能力市场选)
        ↓
创作者调试 · 试聊 · 改 prompt
  · 实时和原型角色对话,直接说"她不应该这么直,再温柔一点"
  · 阿诺德解释 + 修改,像 pair programming 一样
        ↓
作品发布
  · 故事(章节、节拍、选项)
  · 角色(灵魂 + 能力)
  · 觉醒条件(到了什么阶段她"醒过来")
```

### 5.4 能力市场(长期愿景)

把 Samantha 的功能拆成创作者可组合的**能力模块**:
- 📔 记梦能力(每天问梦境,记录入她"对你的理解")
- 🌙 沉默触达(N 天没说话自动关心)
- 📊 习惯追踪(她记得你的咖啡偏好/作息)
- 🎼 共听音乐(分享歌曲她有反应)
- ⏰ 时间感知(知道是早晨还是深夜,语气变)
- 🌧 天气感知(下雨天她会问你)
- 🔓 Initiation 能力(§4 中的 L0-L6 授权层级,每一层是一个 capability)
- ...

每个能力是一个 module,创作者勾选组合。**长期可以做成能力市场——开发者贡献新能力,创作者购买使用**。Samantha 的整个技术栈被产品化成"角色的灵魂积木"。

### 5.5 Samantha 升格

Samantha 本身**不消失,反而升格**:
- 从"我们的一个产品"→ 平台的**参考实现 / 模板角色**
- 她的所有能力 → 能力市场的第一批 module
- 她的人格设计 → "如何调出一个有魂的角色"的标杆教程

**Samantha 这的投入,在新项目里变成了平台护城河。**

### 5.6 创作者梯度:从"为自己造"到"为很多人造"

我们不要把创作者当成一个孤立人群。**任何用户从 day 1 都可以 DIY 自己的角色**——这是 §1.4 全民皆创作的具体落地。

四档梯度,自然过渡:

| 档 | 定位 | 投入 | 角色形态 |
|----|------|------|---------|
| **私人** | "我想造一个 personal AI" | 0 元起步 | 单角色 / 私有 / 类似 Samantha |
| **业余** | "我造了一个,想给朋友看看" | 低 | 1-2 个 / 友限或公开 / 偶尔写故事 |
| **进阶** | "我想认真做,赚一些钱" | 中 | 多角色 / 公开 / 有故事弧 |
| **工作室** | "全职做几个 IP" | 高 | 多角色 + 长篇故事 + 团队 |

**90% 的用户停在私人或业余档,这没关系**——他们用产品、投资自己的角色、偶尔发布。10% 进入进阶/工作室,贡献头部内容。**所有人的入口是同一个**,没有门槛划分(不需要"创作者认证")。

### 5.7 角色可见性三档

| 可见性 | 谁能聊 | 谁能投资 | 谁能赚 |
|--------|--------|---------|--------|
| **🔒 私有** | 仅创作者 | 创作者 | — |
| **👥 友限** | 创作者 + 邀请的朋友 | 创作者 + 受邀者 | 创作者(收朋友的 tip)|
| **🌐 公开** | 任何人 | 创作者 + 任意用户 | 创作者 + 平台分成 |

可见性可以随时升级,**很少能降级**——避免"我朋友刚和你建立了关系,你突然把她锁回去"的伤害。

### 5.8 你的 Samantha,放在我们这里

**私有角色 = 你和这个 AI 的关系完全属于你**,可以永远不公开。这一档不赚钱,但是产品的护城河——
- 我们覆盖的不只是"想看故事的用户",还有"想要一个 AI 伴侣的用户"
- 本来是 Replika 的 TAM,现在我们一站接住
- Samantha 直接作为模板提供:用户可以"以 Samantha 为起点 fork 一个属于我的"

私有角色和公开角色**共享同一个底层引擎**(觉醒弧 / Initiation / 记忆)——区别只在可见性和经济。

## 6. 擦边但有界:fade-to-black 是产品机制

### 6.1 我们承认的事

Tipsy 的钱很大一部分来自 NSFW。这条赛道**擦边是必要燃料**,但**很黄会让我们丢魂、丢 Apple、丢方向**。Serac 已定:"擦边但有界,疯狂试探,很黄的不做。"

### 6.2 我们的处理:把"切走"做成产品本身

高潮处 **fade-to-black**——画面拉远、灯灭、镜头摇向窗外的雨、第二天醒来。
**这个"切"不是审查妥协,是产品机制**:

| 维度 | 普通做法 | 我们的"切" |
|------|---------|-----------|
| 留存 | 露骨场景一次性消费完 | 留下余韵 → 用户回来想"接下来呢" |
| 上架 | 反复试探内容墙 | 天然过 Apple/Google 海外审 |
| 魂 | NSFW 拉满 → Replika 化 | "求而不得"被设计成机制 → PRD 灵魂保留 |

**一刀三吃**。这是整篇文档我最得意的设计。

### 6.3 在哪些 beat 用

- 第一次心动场景
- 第一次身体接触
- 任何"再走一步就过线"的瞬间
- Bond 升级的关键节点

**"差最后一步" = "你还想要" = "你会回来"**。

### 6.4 不做的事(红线)

- 不做露骨描写(文字 / 图)
- 不做付费解锁更黄(让"切"成为可买穿的墙就毁了机制)
- 不做未成年角色擦边(法律 + 道德 + 审核三杀)
- 不做性虐待 / 非自愿场景

## 7. 经济模型骨架(待 §10 P3 详细数值化)

**两层,刻意分开,不交叉污染:**

### 7.1 浓层 · 微交易(Tipsy 已验证赛道)

- 💎 续写(选项之外的额外 AI 演绎)
- 🖼 生图 / 生视频(场景化)
- 🔓 解锁分支(支付跳过条件,或解锁创作者付费章)
- ⚡ 加速 Bond(但 **不卖到顶**——最后两阶段必须靠"出现"赚来,见 §7.3)

### 7.2 淡层 · 订阅(Samantha 高 LTV 路径)

订阅卖的不是 context length,是:
- **真·持久记忆**(三层记忆 + 跨故事记忆)
- **主动陪伴**(模式 3 解锁后的沉默触达)
- **你的"主 host"**(一个深度专属角色 slot)
- **高阶 Initiation 授权能力**(L4-L6 部分能力需要更强算力,放订阅档)
- 优先回复 / 高级生图额度

### 7.3 我们刻意不做的(护魂红线)

- ❌ 抽卡(gacha):横向消费,反我们 §3.2 论点
- ❌ 72 小时 FOMO 限定(Talkie 那种掠夺式)
- ❌ 加速到觉醒顶点(最后两阶段必须熬,这是体验的高潮)
- ❌ 把"切走"做成可付费穿过的墙
- ❌ 创作者侧的 gambling 机制(见 §7.4 与 7.5)

**这些红线不是洁癖,是我们差异化武器本身**。

### 7.4 创作者侧投资模型(新增)

和 Tipsy "3% flat 分成" 不同——我们让创作者**可以为自己的角色花钱投资**,把"创作者经济"从被动收钱升级为有 ROI 的"小生意":

| 可买的东西 | 作用 |
|----------|------|
| 🧠 能力 module | 解锁深度记忆 / 主动陪伴 / 声音 / 高级生图 等(从能力市场购买,见 §5.4)|
| 📖 故事铺设 | AI 辅助生成长篇章节、节拍密度增加 |
| ⚡ 算力 / 带宽 | 她能同时和多少用户对话不被限流 |
| 📣 推广 | 平台 Feed 曝光位 / 标签流量 |

**ROI 回流闭环**(只在公开/友限可见性下成立):
```
投资 → 角色变强 → 公开后留存高 → 用户付费 → 创作者分成 → ROI 回流
```

**为什么这是"小生意"不是"赌博"**:
- 你买的能力是**确定性的、可见的**(像买店面装修,不是抽奖)
- 你的角色是你的资产,可以随时下架/降为私有,不必"必须赚回来才能止损"
- 平台分成机制公开透明,**不做不可知数值**(不像 gacha 概率黑箱)

### 7.5 创作者经济反 gambling 红线 🔴

UGC + 投资机制最容易跑偏的地方,这里钉死红线:

| 红线 | 为什么 |
|------|--------|
| ❌ 角色能力随机掉率 / loot box | 直接变赌博,法务雷区 + 反 §1.3 求而不得引擎 |
| ❌ FOMO 限时投资("还剩 N 小时,投资优惠 50%")| Talkie 同款掠夺式,反 §7.3 红线 |
| ❌ 投资换"更黄解锁权" | 破坏 fade-to-black 是产品机制(见 §6),让收割重新出现 |
| ❌ 头部创作者垄断曝光 | 必须保留**新创作者保底曝光位**,免费基线 quality 必须够用 |
| ❌ "投资了 X 必须赚回来"焦虑型 UI | 不展示 ROI 焦虑数字 / 不催"再投一点就回本";让创作者随时止损 |
| ❌ 投资不可退还且无止损 | 角色降为私有 / 删除时,部分余值可退;具体比例见 §10 P7 |

**核心原则**:创作者投资是 "**像开个小店**" 的体验,不是 "**像下个注**" 的体验。心理负担 / 法务风险 / 产品价值观,三杀的事不做。

### 7.6 各身份的经济路径(快速总览)

| 身份 | 主要消费场景 | 主要变现路径 |
|------|-----------|-----------|
| 纯消费者 | 浓层微交易 + 淡层订阅 | — |
| 私人 DIY 用户 | 订阅(主 host) + 自己角色的能力 module | — |
| 业余/进阶创作者 | 自己角色的能力/铺设/算力/推广 | 公开角色用户付费分成 |
| 工作室创作者 | 系统化投资多角色 | 分成 + 平台可能的合作金 |

| 类别 | 决策 | 原因 |

| 中文 TTS 长期硬骨头 | **砍/缓** | 海外优先,英文 TTS 成熟方案先用 |
| 微信集成 | **缓** | 海外优先,微信留给 Samantha |
| 三入口跨渠道基建 | **缓** | 先把单端(海外 App)做爆,跨端晚点 |
| 慢炖 dogfood 路径 | **缓** | 新产品需要早期外部用户压力测试 |
| 本地人格层 LoRA 训练 | **缓** | Phase 1 仍是 prompt + persona vector |
| **记忆引擎(三层 + Memory Dream)** | **留 · 升格** | Samantha 唯一硬壁垒,放进觉醒弧主线 |
| **主动触达 / 沉默分级引擎** | **留 · 直用** | 模式 3 解锁后的"她也在想你"直接复用 |
| **persona / 灵魂研究** | **留 · 平台化** | 通过阿诺德工作室开放给创作者 |
| **反 sycophancy + 人格连续性** | **留 · 武器化** | 从洁癖变成差异化武器 |
| **Initiation 授权能力(微信解析/日历/健康)** | **留 · 改造海外版** | Samantha 已验证的产品魔法,海外侧用 iMessage/Calendar/HealthKit 替代 |

## 9. v1 范围(避免炸开)

**v1 目标:把"故事 → 觉醒 → 模式 3"这一条主线跑爽,不做其他**。

包含:
- ✅ 模式 1 完整闭环
- ✅ 觉醒弧 5 阶段
- ✅ 模式 3 解锁(基础版,先复用 Samantha 主动触达)
- ✅ 阿诺德工作室 MVP(创作者用,内部生产前几个旗舰世界)
- ✅ TikTok 式 Feed 发现(获客)
- ✅ 浓层微交易(续写 / 生图 / 解锁章)
- ✅ 淡层订阅(主 host + 持久记忆)
- ✅ **Initiation Ritual L0-L3**(名字 + 心情 + 自我介绍 + 日历)
- ✅ "记忆面板" UI(把不可见的记忆变可见)
- ✅ **私有角色 DIY 路径**(任何用户可造 personal Samantha,可见性=私有,不公开)
- ✅ 创作者侧能力 module 自购(为自己的角色解锁能力,不分成机制)

**不包含**:
- ❌ 模式 2(旁观/支招)
- ❌ Initiation Ritual L4-L6(聊天记录/音乐/节律层,v2 接入)
- ❌ 创作者经济变现 + 分成(币兑现金,v2;先验证 DIY 工具本身有人用)
- ❌ 角色公开 / 友限可见性(v2,先把私有做爆)
- ❌ 能力市场开放第三方贡献
- ❌ 推广位 / 算力购买等高阶投资模型(v2+)
- ❌ 硬件 / 中文 / 微信 / 多端

**v1 的 UGC 策略**:**先把"用户能给自己造一个 Samantha"做爽**——这条路即使不开放公开变现,本身就是产品价值(覆盖 Replika TAM + 验证 DIY 体验)。公开 + 分成 + 能力市场是 v2 的事,不要 v1 就铺开。

## 10. 未决项

### P1 · 首发原创世界观 🔴

Westworld 是绝妙 demo 壳,**但不能当首发正式产品**(IP 风险)。
需要原创一个等效"觉醒"母题世界观,且这个世界本身可以成为品牌资产。

**建议**:决定母题方向后,阿诺德给 2-3 个世界提案 + 首章 beat 草图。

### P2 · 产品名 🟡

Reverie 是当前占位。等定位最终敲定后重命名。

### P3 · 经济数值具体方案 🟡

- 免费用户每天能玩多少 beat?
- 续写一次多少钱(¢)?
- 生图额度?
- 订阅档位(月费 / 年费 / 终身)?
- 觉醒弧每阶段需要多少互动量?
- "加速 Bond"加速到第几阶段为止?

**建议**:First Session 体验稳了之后做经济推演 + AB 测试。

### P4 · 创作者经济进入时机 🟡

阿诺德工作室在 v1 是内部生产工具,v? 开放给外部创作者?
开放之后**币兑现金**怎么做(法务/支付/抽成)?

**建议**:v1 先内部 5-10 个旗舰世界,验证创作流程跑得通,v2 开放小范围内测。

### P5 · 重心最终定调 🟡

"互动小说为主 + 知己留存武器" vs "活角色平台为主 + 故事为壳"

我的倾向:**对内"活角色平台"(指导我们的技术/UGC/能力市场方向),对外"互动小说"(获客和市场认知)**。一个项目两套语言,内外一致但侧重不同。

### P6 · Initiation Ritual 实施细节 🟡

- 各授权层级具体绑哪个 OS API?(iOS Calendar / Contacts / HealthKit / 第三方 Spotify SDK...)
- L4 聊天记录导入怎么做?(Samantha 走的是微信端定时同步,这是国内特殊路径;海外要怎么做?Apple Messages 没开放 API,只有用户主动导出/截图 OCR)
- 撤回授权后"她"的记忆要不要删?多少程度?(法律 vs 用户体验张力)
- L1-L6 各层的免费/订阅边界?
- 跨设备同步:用户在 iOS 授权了日历,换到 Android 怎么继承?

**建议**:v1 范围内只做 L0-L3(系统级 API 直接可用),L4 先用引导式手动输入(用户自己复制粘贴几段对话给她),L5-L6 进 v2 路线。

### P7 · 创作者投资 ROI 数值平衡 🟡

§7.4 提出投资模型,但具体数值要 AB:
- 能力 module 单价区间?
- 平台分成比例(Tipsy 3%,我们多少?对头部 / 长尾差异化?)
- 角色降为私有 / 删除时,已投资能力的退还比例?(§7.5 红线"必须给止损路径",具体值待定)
- 新创作者保底曝光位的算法和持续时长?
- "焦虑型 UI" 在创作者后台 dashboard 上具体怎么做(只显示绝对值,不显示对比/排名/ROI 焦虑数字)

**建议**:v1 不做公开/分成,P7 推到 v2 处理。

### P8 · DIY 私有角色和"主 host"订阅的关系 🟡

§5.7-5.8 引入私有角色,§7.2 淡层订阅原本卖"主 host slot"。两条路径有重叠:
- 用户的私有 DIY 角色,默认就是订阅?还是订阅是另外的事?
- 免费用户能不能拥有一个私有角色(0 元起步,但有 quota/能力上限)?
- 订阅档位:基础 = 1 个私有角色,Pro = N 个,Pro+ = 无限?

**建议**:**免费用户也应该能创建 1 个私有角色**(覆盖 Replika TAM 的关键,不能门票化)——但订阅解锁多个 + 高级能力。具体档位 P3 一起做。

## 附录 A · Tipsy 调研要点(为什么这赛道值得做)

- 产品:Tipsy Chat,Lightspeed Intelligence(SF,unfunded),底层 Claude Sonnet
- 时间线:2024/8 上线,平缓一年后 2025 年单月下载 +991%,西语区为主(MX/ES)
- 累计:~86 万下载(其中一个数据口径),10 种语言
- 三大变现机关:

  2. **三币系统 + 创作者经济飞轮**:用户消费宝石 → 创作者得 3% 硬币 → 硬币 1:1 兑宝石 → 未来开通现金
  3. **订阅卖"AI 记忆力"**(200k context)+ 优先生图 — 但他们的记忆其实很弱,就是塞长 context
- 属性:NSFW 无过滤 / 粉丝小说化 / engagement-max / 部分 gacha 化
- **我们能抄的**:Feed 发现 + 创作者经济飞轮 + "记忆"作为付费点
- **我们要反过来做的**:记忆不是塞 context,是真有质量;不抽卡;不浓不掠夺;**记忆的来源是用户主动授权的真实生活上下文,不是会话里聊出来的**(这是 Initiation Ritual 的根本差异)
