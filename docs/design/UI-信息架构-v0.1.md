## 1. 底部导航栏 · 5 tab 定案

```
┌──────────────────────────────────┐
│                                   │
│         (page content)            │
│                                   │
├────────┬────────┬────┬────────┬──┤
│  🔥    │  🧭   │ ➕ │  💬 ●  │ 👤│
│ 首页    │ 探索   │创作 │ 消息   │ 我│
└────────┴────────┴────┴────────┴──┘
```

| Tab | 作用 | 内容 |
|---|---|---|
| 🔥 首页 | Feed 流沉浸推荐 | TikTok 式:一屏一个作品 30-60s 预告,上下滑动,零摩擦发现 |
| 🧭 探索 | 主动浏览 | 网格 + 分类(时代 / 关系 / 时长 / 尺度) + 搜索,像 Xiaohongshu 探索页 |
| ➕ 创作 | 阿诺德工作室 | 造私有作品 · v1.5 §4.3 消费→创作转化飞轮的入口 |
| 💬 消息 | 我的所有关系 | 玩过 / 正在玩的作品 + 已解锁的角色私聊,见 §2 |
| 👤 我 | 个人 | 订阅 / 积分 / 收藏 / 设置 |

**首页 vs 探索的分工**:
- 首页 = 被动被推(沉浸,3 秒决策)
- 探索 = 主动找(浏览,自选)
- 两者都可以进新作品,但目标不同

**"从发现到消息"的搬迁逻辑**(参考微信通讯录 vs 聊天):
- 首页/探索里的作品是**推荐池**(还没开始玩的)
- 一旦点进作品并开始玩,自动**搬到消息 tab**,首页/探索不再重复推
- 从消息 tab 可以随时回访已玩的作品

## 2. 消息 Tab · 三视图

**核心判断**:用户心智单位是"**角色/作品**",不是"消息类型"。所以顶层是**同一个 flat list**,只是角色 vs 故事**样式不同**;顶部 3 sub-tab 是**筛选器**。

### 2.1 顶部 3 sub-tab

```
┌──────────────────────────────────┐
│         💬 消息                   │
├──────────────────────────────────┤
│  [All]   Characters   Story       │
├──────────────────────────────────┤
```

- **All** · 默认 · 角色和故事按最新活动时间戳混合排序
- **Characters** · 只显示已解锁通道的角色(⭕ 圆头像 list)
- **Story** · 只显示所有玩过的故事(▭ 方形封面 list · 已解锁+未解锁都在)

### 2.2 All tab · flat list 混合排序

```
┌──────────────────────────────────┐
│  ⭕ Lucian                 2m ●  │  ← 角色(已解锁)
│     "米兰在下雨。"                │
│                                   │
│  ▭ 白月光·初二             5m    │  ← 故事(未解锁通道)
│     "他把画布搬到窗边..."          │
│                                   │
│  ⭕ Iris                    1h   │  ← 角色
│     "又是那个坐 3 号桌的?"        │
│                                   │
│  ▭ 合同新娘 · 试玩中        3h   │  ← 故事(试玩中)
│     "你穿上礼服,他背对你..."       │
│                                   │
│  ▭ 蓝月亮 Iris              1d   │  ← 故事(已解锁 · 有对应角色 Iris 在上面)
│     "你点了两杯 Blue Moon..."      │
│                                   │
└──────────────────────────────────┘
```

**关键 · 已解锁的作品会同时出现两行**:一个 ⭕ 角色行(私聊入口) + 一个 ▭ 故事行(继续玩剧情)。它们**共享同一份角色记忆**,只是接触方式不同。用户不觉得重复,反而觉得"我可以选择跟她聊,还是回去那天"。

### 2.3 排序规则 · 微信式

- 谁最新有活动就置顶
- 未回复的话被更新的推下去一级
- 时间戳右上角
- 未读小红点

**AI 主动来消息**(v1.5 §4.3):不扣积分 · 自动置顶 · 有光圈闪一下

### 2.4 角色行样式 · ⭕

```
┌──────────────────────────────────┐
│  ⭕                                │
│  【圆头像】 Lucian              2m │
│           "米兰在下雨。"      ●●● │
└──────────────────────────────────┘
```

- **⭕ 圆头像**:强调"人"
- 名字**大字号**
- 最新私聊气泡预览(截断到一行)
- 未读点堆叠
- 时间戳右上

### 2.5 故事行样式 · ▭

```
┌──────────────────────────────────┐
│  ▭▭▭                              │
│  ▭▭▭  白月光·初二              5m│
│  ▭▭▭  "他把画布搬到窗边..."       │
│       WHERE YOU STAND: Unfinished │
└──────────────────────────────────┘
```

- **▭▭▭ 方形封面**:强调"内容"
- 作品名
- 最近故事内容 preview(参考 Claire wireframes · 用故事里最后一段 narration 或 speech,不显示 Beat 编号 · v1.5 §3.2)
- **WHERE YOU STAND: Unfinished** — 状态标签(不是进度百分比)
- 时间戳右上

**关键约束**:
- ❌ 不显示 Beat 几 / 章节几
- ❌ 不显示进度百分比条
- ✅ 用状态标签(Unfinished / Ongoing / 通道已开启 等)
- ✅ 用文本 preview 让用户想起自己上次玩到哪

## 3. 进入作品 · 3 Tab 上限

一个作品最多 3 tab:**故事 + 主角色 + 副角色**。副角色**创作者选是否开放**(v1.5 §6.2 创作者标准)。

### 3.1 进入已解锁作品(带 3 tab 上限)

```
┌──────────────────────────────────┐
│ ← 🎨 油画老师                     │
├──────┬────────┬─────────────────┤
│ 故事  │ Lucian │  沐的闺蜜(可选) │
├──────┴────────┴─────────────────┤
│                                   │
│  (对应 tab 的内容)                │
│                                   │
└───────────────────────────────────┘
```

- **故事 tab** — 剧情推进(Cinematic 场景 · Suggested 选项 · 见 §5-6)
- **主角色 tab** — 与主角色的私聊(Lucian)
- **副角色 tab** — 与副角色的私聊(创作者选择开放,通常是闺蜜/师父/同伙,**不是主角色的敌人/情敌**)

### 3.2 进入未解锁作品(只有故事 tab)

```
┌──────────────────────────────────┐
│ ← 📖 白月光·初二                  │
├──────────────────────────────────┤
│                                   │
│  故事时空(single)                │
│                                   │
└──────────────────────────────────┘
```

不显示假 tab(不显示灰的"私聊"tab) · 保留通道开启的仪式感。

### 3.3 进入角色(反向跳转)

从消息 tab 点角色 ⭕ Lucian:

```
┌──────────────────────────────────┐
│ ← ⭕ Lucian                       │
├───────────┬─────────────────────┤
│  💌 私聊  │  📖 故事《油画老师》 │
├───────────┴─────────────────────┤
│                                   │
│  Lucian: 米兰在下雨。              │
│  Lucian: 你在做什么。              │
│  [输入框]                          │
│                                   │
└───────────────────────────────────┘
```

顶部 tab 有 "→ 故事" 一键跳转 · 不用退回消息 tab 中转。

## 4. 6 类 Beat 视觉规范(Claire wireframes vocabulary)

**规范来源**:Claire 的 wireframes 定义了 6 类 beat 的 UI 分工。团队开发时对着这 6 类做,视觉一致性有保证。

### 4.1 6 类 + 1 类清单

| # | Beat 类型 | 用途 | 视觉规范 |
|---|---|---|---|
| 1 | **SCENE MARKER** | 时间 / 地点 / 章节 | Mono 字体 · Hairline 细边 · 大写 · 无引号 |
| 2 | **NARRATION** | 旁白 · 世界和当下 | 居中衬线体 · 无引号(不是说出口)· 描写场景/心情 |
| 3 | **CHARACTER SPEECH** | 角色说话 | 左气泡 + avatar · Sans · 「引号」 |
| 4 | **YOUR ACTION** | 你的动作 · 未说出 | 旁白语气 · 暖色 tick · 「你做了什么」 |
| 5 | **YOUR SPEECH** | 你说的 · 说出口 | 右气泡 · 你的暖色 · 「引号」 |
| 6 | **INNER THOUGHT** | 内心独白 · 未闻 | ✦ 标记 · 暗色衬线 · 私密感 |
| + | **CHOICE** | 决策点 | 3 suggested 卡片 + Free-write toggle |

### 4.2 一段示例(Beat 2 · 期中夜)· 展示 6 类混排

```
━━━ SCENE MARKER ━━━
两周后 · 期中夜 · 画室 · 22:30

━━━ NARRATION ━━━
期中考核。你的画你觉得不够,磨到深夜。

━━━ CHARACTER SPEECH (Lucian) ━━━
「晚了。」

━━━ CHOICE ━━━
▸ 「……对不起。」(默认 · 顺从的沐)
  「我想画完再走。」
  「您也没走。」
  或写你自己的回复...

━━━ INNER THOUGHT ━━━
✦ 他记得。
✦ 他记得我不敢抬头。

━━━ YOUR ACTION ━━━
你放下书包,走过去。
```

**每一类 UI 分开表达**,同一场景的 6 种"声音"永远不混。这就是 Claire 说的 *"Six voices, never mixed"*。

## 5. Suggested ↔ Free-write · 回复机制

Claire wireframes 的核心交互 · 同一个 send button 驱动两种输入模式。

### 5.1 Suggested 模式(默认)

```
━━━ CHOICE ━━━
┌─────────────────────────────┐
│ ▸ 「……对不起。」  (默认)      │
│ 「我想画完再走。」            │
│ 「您也没走。」                │
└─────────────────────────────┘
    or write your own... ▼
```

- **3 个 suggested 卡片** · Chip 样式
- **默认第一个最刺激/推进关系深** · 承接 v1.5 §1.5 下沉定位
- 用户点即用 · 极低摩擦

### 5.2 Free-write 模式(切换)

点 "or write your own..." 切到自由输入:

```
━━━ CHOICE ━━━
┌─────────────────────────────┐
│  [Free-write 文本输入框]     │
│                              │
│                              │
└─────────────────────────────┘
    ← back to suggested ▲
     [send]
```

- 用户可自由打字扮演沐
- **同一个 send button** 驱动
- 用户输入内容进入她/他的记忆系统(v1.5 §2.1.4 trait 雕刻)

### 5.3 v1 不做的

- ❌ **as another character**(以其他角色身份说话)· Claire wireframes 里有 "Direct scene / Direct others / + who?" · Serac 明确砍
- ❌ 用户自由指挥剧情走向 · v1 只做用户扮演自己

## 6. 通道开启 · UI 事件全流程

这是产品最核心的仪式时刻 · 承接 v1.5 §2.5 · UI 设计权重最高。

### 6.1 触发序列

```
[故事 Beat 8 · 展览夜]
    ↓
用户翻过画布,看到铅笔字"L."
    ↓
━━━ 全屏动画:画布铅笔字慢慢显现 ━━━ (2s)
    ↓
━━━ 手机震动 · 全屏黑色遮罩 ━━━ (1s)
    ↓
弹出文本:「你收到了 Lucian 的第一条消息」
    ↓
[打开] 按钮
    ↓
━━━ 首次进入私聊界面 ━━━
    Lucian: 米兰在下雨。
    Lucian: 你在做什么。
    ↓
用户回复或退出
    ↓
━━━ 退出到消息 tab ━━━
《油画老师》升级 · Lucian 角色行**从头出现**,置顶,金光扫过动画
```

### 6.2 通道开启前后的视觉差异

**通道开启前**(Beat 8 及之前):
```
▭▭▭  油画老师              5m
▭▭▭  "他把画布搬到窗边..."
```

**通道开启后**(Beat 8 尾 + 之后):
```
╔═══════════════════════════════╗
║ ⭕ Lucian              2m ●   ║  ← 金边卡片 · 置顶
║   "米兰在下雨。"               ║
╠═══════════════════════════════╣
║ ▭ 油画老师              5m    ║  ← 故事行仍在
║   "画布背后铅笔字..."           ║
╚═══════════════════════════════╝
```

同一作品**两行都在**:一个是角色私聊入口(⭕),一个是故事回访入口(▭)。它们**共享记忆**,只是接触方式不同(v1.5 §2.3)。

### 6.3 后续 · AI 主动来消息的置顶

Lucian 主动来消息:
- 该角色行自动置顶
- ⭕ 头像有光圈闪一下(0.5s)
- 未读小红点
- **不扣积分**(v1.5 §4.3 关键设计)

## 7. Cinematic 沉浸方向定案

**从 Claire wireframes 三选一**(Cinematic / Editorial / Scene cards)· 我们选 **Cinematic**。

理由:
- **旁白浮在场景上** · 分镜感 · 沉浸最强
- 匹配 v1.5 §1.5 下沉定位("首屏要劲爆,不能纯氛围")
- 视觉能力最强 · Dreamina 出图/视频天然适配
- **反糖水粉色少女漫** 美学承接油画老师故事

### 7.1 Cinematic 单 beat 布局

```
┌──────────────────────────────────┐
│ [全屏场景图 · Dreamina 生成]     │
│ [暗色调 · 分镜感]                 │
│                                   │
│              ━━━ SCENE MARKER ━━━ │
│              两周后 · 22:30       │
│                                   │
│         ━━━ NARRATION ━━━        │
│         期中考核。你的画你觉得    │
│         不够,磨到深夜……          │
│                                   │
│  [左气泡 Lucian avatar]           │
│  ┌──────────────┐                │
│  │「晚了。」      │                │
│  └──────────────┘                │
│                                   │
│                                   │
│  ┌──────────────────────────────┐│
│  │ ▸ 「……对不起。」             ││
│  │   「我想画完再走。」          ││
│  │   「您也没走。」              ││
│  │   or write your own... ▼    ││
│  └──────────────────────────────┘│
└──────────────────────────────────┘
```

- **场景图**:全屏铺底 · Dreamina 生成 · Beat 视觉 prompt 见 [故事文档](./故事母题与首批构思-v0.1.md) §5.7
- **旁白/NARRATION**:居中衬线 · 半透明浮层
- **对话气泡**:局部 · 玻璃质感(Claire 的 *glass dialogue*)
- **Choice 卡片**:浮在最下方 · 3 个 + Free-write toggle

### 7.2 视觉基调

参考:
- *Call Me By Your Name*(意大利田园禁忌感)
- *Portrait of a Lady on Fire*(古典油画光)
- *爱在黎明破晓前*(1990-2000 现代文艺)

**反对**:Tipsy 式糖水粉色 · 少女漫风 · 高饱和 · 过度美颜。

## 8. Character-first opening · 替代书封面

Claire wireframes 的开场设计 · 我们采纳。开场围绕**角色 + 关系 + 场景锚点**,不是作品封面。

### 8.1 开场屏布局

```
┌──────────────────────────────────┐
│  [Lucian 全身像 · 深绿毛衣 · 背光]│
│  [Dreamina 生成 · 高分辨率]      │
│                                   │
│       You're about to meet        │
│      "你从没爱过的第一个人"        │
│                                   │
│      艺术学院 · 秋 · 画室          │
│      Chapter 1 · 第一堂课          │
│                                   │
│  35 岁 · 意大利留学派 · 已婚      │
│  他是你大一油画课的老师。          │
│  但他看你的时候,不像看学生。      │
│                                   │
│  WHERE YOU STAND: Not yet begun   │
│  [ Enter the story ▸▸ ]           │
│                                   │
│  A.I. character · fiction only    │
└──────────────────────────────────┘
```

包含:
- **角色艺术图**(全屏)
- **You're about to meet + 关系锚点**(一句话形容他)
- **世界锚点**(时间 / 地点 / 章节)
- **一段简介**(2-3 行 · 交代年龄 / 身份 / 关系 tension)
- **WHERE YOU STAND: Not yet begun**(状态,不是进度)
- **[ Enter the story ▸▸ ]** — CTA
- **AI 免责**(法务基线)

### 8.2 已开始玩过的作品 · 状态更新

用户玩到 Beat 3 后再打开这个作品:

```
WHERE YOU STAND: Unfinished
[ Continue the story ▸▸ ]
```

CTA 从 "Enter" 变成 "Continue"。

### 8.3 通道开启后的作品

```
WHERE YOU STAND: 通道已开启 · 与 Lucian 通信中
[ Continue the story ▸▸ ]
[ Chat with Lucian → ]
```

给两个 CTA · 用户可以选择回故事或去私聊。

## 9. Feed 流(首页)· 30-60s 短视频钩子

承接 v1.5 §1.5 下沉定位 · **首屏必须 3 秒定生死**。

### 9.1 卡片规格

- **30-60 秒竖屏视频** · 抖音尺寸 9:16
- **首 3 秒**:角色魅力 shot + 关系张力锚点 + 悬念/擦边暗示
- **字幕**:一句让用户想点进的 tagline(参考"有人系着你,你系着谁?"·"这不该发生,但已经发生了")
- **底部 CTA**:[ Enter the story ] · 半透明

### 9.2 举例 · 油画老师 Feed 卡片

```
[Beat 7 · 湿漉漉白衬衫的 Lucian · 台灯黄光]
[他跪在你面前 · 手在你脚踝]
[8 秒]
    ↓
字幕:"「你是我的学生。」"
[5 秒]
    ↓
字幕:"但他没有松手。"
[3 秒]
    ↓
[画面 fade to black]
[10 秒静音]
    ↓
[晨光 · 便条特写]
"你睡的真香。晚点回来。"
[8 秒]
    ↓
CTA:[ 从头开始这个故事 ▸▸ ]
```

**总时长约 34 秒** · 首 8 秒定生死。

## 10. 附录 · 术语速查

| 术语 | 定义 |
|---|---|
| **故事时空** | 剧情推进的 tab · Cinematic 分镜 · 6 类 beat 混排 |
| **日常时空** | 通道开启后的私聊 tab · 短对话 + 时间戳 |
| **通道开启** | 关系升级的仪式时刻 · UI 事件见 §6 |
| **Cinematic** | 3 个沉浸方向中选定的 · 旁白浮场景上 · 分镜感 |
| **Suggested / Free-write** | 回复的两种模式 · 同一个 send · 见 §5 |
