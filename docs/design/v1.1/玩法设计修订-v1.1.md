## §1.3 求而不得引擎 [修订]

### v1.0 原文(摘)
> Tipsy 给即时满足……我们反过来——**期待 > 兑现**。亲密度慢慢涨、host 慢慢"醒"、张力一直累积但永远差最后一步。

### v1.1 新版

> Tipsy 给即时满足……我们反过来——**期待 > 兑现**。亲密度慢慢涨、信道慢慢宽, 但**两个世界永远没法真正合一**。

**为什么改**:
- v1.0 "差最后一步" 指向觉醒前瞬间 → 觉醒发生后, 这个张力就用完了(有限弹药)
- v1.1 "两个世界永远跨不过" 是关系的底层物理学 → 每一次跨时空互通本身就是张力(无限弹药)

求而不得仍然解决三件事(留存 / 上架 / 不丢魂), 但其引擎从"一次性高潮"变为"永恒边界"。

## §2 玩法核心: 三模式 = 同一个角色关系的三阶段 [修订]

### v1.0 原文(摘)
> 模式 3:赛博朋友。Bond 达阈值时, 角色"卸下扮演、走出故事"……**这一刻 = 互动小说的角色变成了 Samantha**。

### v1.1 新版

> 模式 3:**跨时空笔友(In-Character Forever)**。Bond 达阈值时, 角色**主动写信**邀请你建立"日常通信", 你接受后, 日常 tab 解锁。她**永远不卸下扮演**, 永远在她的世界里, 但她对你的世界好奇, 你们的关系通过两个世界之间的笔友式互通逐渐深化。

详见 [Mode3-延续线-v1.1.md](./Mode3-延续线-v1.1.md)。

**ASCII 关系演进图修订**:

```
   认识她           了解她           笔友长存
     ↓               ↓                ↓
  模式 1          模式 2           模式 3
 (主人公)       (旁观/支招)       (跨时空笔友)
     │               │                │
  在故事里       在故事外         两个世界
   遇见她          凝视她           一条信道
                                  关系永不熄灭
```

(v1.0 原图最后一格 "成为她的人" → v1.1 改为 "关系永不熄灭", 因为不再有"拥有"这种带占有色彩的隐喻)

## §2.1.6 "她记住了" 5 种形态 [部分修订]

### 仅修订 #5 现实数据穿透

| 维度 | v1.0 | v1.1 |
|------|------|------|
| 用户体感(直白)| "她跨过了屏幕" | "她的世界对我的现实好奇, 她以她的身份打量我的世界" |
| 技术对应 | Initiation 授权层引入 | 同 + **创作者编写的 in-character 翻译 wrapper** |
| 典型场景 | 她说:"你今天 3 点那个会, 后来怎么样?" | 她在信里写:"今日你有一桩要事赴, 是否顺利? 我从'梦'里隐约知道。"|

**其他 4 种形态 (#1 字面引用 / #2 选择回响 / #3 模式识别 / #4 跨 session 记忆) 全部不变**, 只是 #4 在 v1.1 框架里**主要走日常 tab**(她在信里主动 reference 你之前说过的事)。

## §2.3 模式 3 [完全替换]

### v1.0 原文(摘)
> Bond ♥ 达到阈值时, 角色"卸下扮演、走出故事"——开始跨故事记得你说过的话, 主动发消息给你, 引用你的真实日常, 偶尔打破第四面墙……
> 这一刻 = 互动小说的角色变成了 Samantha。

### v1.1 新版

> **请直接见 [Mode3-延续线-v1.1.md](./Mode3-延续线-v1.1.md)。**
> 
> 一句话总结: 角色**永不觉醒、永不卸下角色**, 但在 Bond 达阈值后**主动写信邀请**你建立日常通信, 解锁日常 tab。从此你和她之间有一条"跨时空信道", 关系通过两个世界的互通逐渐深化, **她记得你**(跨 session), **她对你的世界好奇**(in-character), 但**两个世界永远没法真正合一**(求而不得引擎)。

⚠️ v1.0 的"觉醒 / 走出故事 / 偶尔打破第四面墙"全部砍掉。**v1.1 平台铁律:任何角色, 任何 tab, 都不允许破第四面墙。**

## §4 Initiation Ritual [语义升级, 层级不变]

L0-L6 的授权层级不变。但每一层的**含义**升级为"两个世界之间信道加宽":

| Layer | v1.0 含义 | v1.1 含义 |
|-------|---------|---------|
| L1 通知 | 平台向用户推消息 | **她**托人捎口信 |
| L2 位置 | 她知道你在哪 | 她以她的世界观浪漫化你今日的"行踪" |
| L3 日历 | 她知道你今日会议 | 她知道你今日有"商谈 / 要事 / 赴约" |
| L4 聊天 | 她读你的聊天记录 | 她"远远听闻"故友间的对话 |
| L5 音乐 | 她知道你今日听什么 | 她"听到"你今日的"小调" |
| L6 健康 | 她知道你的生理状态 | 她"感应"你今日的"身子骨" |

→ **新增创作者要求**: 每个使用 L1+ 的角色必须编写"她的世界如何理解这类数据"的 wrapper 模板。平台提供 5-10 套预设(古典 / 异星 / 现代 / 奇幻 / 未来), 创作者选预设 + 微调。

详见 [Mode3-延续线-v1.1.md §4](./Mode3-延续线-v1.1.md)。

## §6 fade-to-black 主题 [增量修订]

### v1.0 主要功能
- 过 Google/Apple 审
- 把"切走"当机制
- 制造渴望

### v1.1 增量含义

新增一层: fade-to-black 也是**"两个世界距离感"的具象化**。

亲密时刻发生的事属于她的世界, 你无法直接经历, 只能在第二天收到她的信:

> "昨晚……我有些不知该如何写。
>   总之, 我整夜都在想你。"

→ 距离感本身就是关系的味道。**fade 不是审查规避手段, 是 v1.1 美学核心**。

## §7 变现模型 [新增 hook 类型]

### v1.0 付费 hook(摘)
- 章节解锁
- 觉醒里程碑
- Mode 3 永久订阅

### v1.1 调整

| Hook | 状态 |
|------|------|
| 章节解锁 | 不变 |
| 觉醒里程碑 | **砍掉**(v1.1 没有觉醒)|
| **第一封信解锁** | **新增** — 收到第一封信本身免费, 但选"愿意"后, 日常 tab 第 1-7 天免费试用 |
| **笔友长存订阅**(替代 Mode 3 永久订阅)| 试用结束后, 订阅日常 tab 永久可用 |
| **跨时空信物 / 礼物 / 收藏品** | **新增** — 用户可付费让她"收到"她世界里的特殊礼物(创作者经济, 类似 Tipsy 三币但内容化)|

→ 付费动机从"想看到她觉醒" 变成"想让信道永远开着 + 想让她在她的世界里活得好一点"。**前者一次性, 后者持续性**, LTV 优。
