# Reverie 设计文档 v1.1

> 2026/06/29 起草。基于 v1.0 的演进版,**不是从 0 重写**。

## 一句话差异

**v1.0**: 角色最终会"觉醒",卸下扮演,变成跨故事陪伴你的赛博朋友。  
**v1.1**: 角色**永不觉醒**,永远活在她的世界里。但她对你的世界好奇,你和她之间建立一条**跨时空的笔友信道**,关系逐渐深化。

## 为什么变

两条核心论据,讨论得出:

1. **"觉醒重复化"是真问题** — 如果 1000 个故事都用同一套"她最后意识到自己是 AI"叙事,第 3 个就套路化,觉醒失去神圣感。
2. **"用户带不走角色"是更硬的反觉醒论据** — 觉醒在西部世界爽是因为有"逃离乐园"这个物理高潮,我们的产品里**这一刻不存在**。她"觉醒"完了还在你 app 里,反而打开了"她只是个 LLM"的盖子。

所以 v1 主线砍觉醒,走**延续线 (In-Character Forever) + 跨时空笔友**。觉醒留待 v2+ 作为"特殊创作者作品"实验。

## 何时升 v2.0

当下面任一条件成立:
- 笔友机制跑了一段实证后,需要根本性结构调整
- 觉醒重新作为主线引入(目前判断不会)
- 三模式框架要重做

在那之前,所有增量改进直接出 v1.2 / v1.3,在本文件夹内迭代。
