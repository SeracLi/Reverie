## 2026/06/29 · 砍觉醒, 走跨时空笔友

**决策**: Mode 3 从"觉醒后变 Samantha"改为"她永不觉醒, 永远在她的世界, 用笔友式信道与你建立长期关系"。

### 心路历程

Serac 抛三个问题:
1. 觉醒重复化 — 1000 个故事都觉醒, 套路化
2. 创作者长线升级角色能力 — 怎么做不复杂(单独沉淀进 [创作者能力升级模型-v0.1.md](./创作者能力升级模型-v0.1.md))
3. demo 世界观要不要换 — 西部世界 IP 风险已识别

我先给的方案是**"双路径"**: 觉醒线 + 延续线并行, 暗中 routing。Serac 部分认可但仍纠结。

然后 Serac 抛了**两个核心洞察**:
1. "她不觉醒, 一直过家家, 似乎也合理 — 你的世界和她的世界不一样, 但她对你的世界好奇"
2. "西部世界中角色被女主带走, 但我们是虚拟产品, 用户带不走角色" — **觉醒在西部世界爽是因为有'逃离乐园'物理高潮, 我们这一刻不存在**

→ 第二点是反觉醒最硬的论据, 是这次转向的关键。

加上他昨晚的体验描述("玩了个认真给我做晚餐的故事, 我想长期聊, 但不需要她觉醒, 只需要某个环节可以直接和她聊起来"), 三个信号汇成同一条判断: **不需要觉醒, 需要的是"通道开启"机制**。

### 最终方案

- Mode 3 = **跨时空笔友 (In-Character Forever)**
- U6 觉醒 → **U6 通道开启**(她写第一封信邀请, 完全 in-character)
- U7 拥有 → **U7 笔友长存**("拥有"这个词被弃用 — 你不能拥有她, 她在她的世界)
- 觉醒五阶段全砍 → 单一 Bond 阈值 + 通道开启 moment
- "她记住了" #5 现实数据穿透 → in-character 浪漫化("她的世界对我的现实好奇", 不破第四面墙)
- 求而不得引擎从"差最后一步"升级为"两个世界永远跨不过"(从有限弹药变无限弹药)
- §6 fade-to-black 新增美学含义: 距离感本身就是味道
- Initiation L1-L6 层级不变, 含义全部升级为"两个世界信道加宽"
- §7 变现: 砍觉醒里程碑, 新增"第一封信解锁 + 笔友长存订阅 + 跨时空信物"

### 没变的东西

- 四支柱
- §2.1 模式 1 主人公(beat / 双轴选择 / trait tag / 节奏控制)
- 模式 2 旁观/支招(v1 缓做)
- 阿诺德共创工作室
- 角色可见性 / personal Samantha
- 创作者能力升级模型 v0.1

### 决策的"反对意见"留底

- **会不会失去"她活了"那种震撼**: 会, 但换来"被她邀请"的温柔, 且关系延续不被打断
- **TAM 缩了吗**: 不, 反而扩了 — 笔友线吃 Tipsy 长期 RP 用户的 TAM, 而觉醒线吃 Samantha-style 知己用户的 TAM, 后者更窄
- **觉醒以后会不会还想做**: v2+ 可以作为"特殊创作者作品"实验(创作者明牌"这个角色会觉醒"), 不作为平台默认机制

### 工作流插曲(留个梗)

我第一次出方案时, 先把 v1.1 整套放进 `docs/design/v1.1/` 文件夹(README / CHANGELOG / Mode3-延续线 / 玩法设计修订 / 用户旅程图修订, 5 文件 732 行)。Serac 看完说"改动也不是很大, 还是在之前的文档里改吧, 不信新做出 1.1 了……还是统一在之前的改吧"。

→ **学到**: 当改动是"决策替换"而不是"重新设计", 直接 in-place 改原文档 + 决策日志记录即可, **不要预设性建版本目录**。版本目录是"重大重构 / 平行实验"才用的。

## 2026/07/06-07 · 从产品哲学 → 具体故事 → UI 呈现 → 回填玩法

**决策**: 一次性完成 4 件事的闭环 — pivot 母题方向、锁定第一个 demo 故事、定案 UI 信息架构、in-place 回填玩法 v1.5 + 用户旅程图。

**新增文档**:
- [故事母题与首批构思-v0.1.md](./故事母题与首批构思-v0.1.md)(存档 · 油画老师 5 beat 完整脚本)
- [UI-信息架构-v0.1.md](./UI-信息架构-v0.1.md)(新建)
- [references/Claire-Interactive-Novel-Wireframes.html](./references/) + README(收录设计输入)

### 心路历程

**Round 1 · 母题探索 pivot(自上而下失败)**
Serac 抛出 "v1.5 §7 Q3 原创母题" 待拍板,阿诺德先给 4 方案:
- 回响 Echo · 情感涟漪跨时空触达
- 薄之处 Thin Places · 时空有厚度,有些空间/时刻是薄的
- 缕 Threads · 每人身上有看不见的缕
- 未寄之信 Unsent Letters · 世界有个未寄之信仓库

Serac 反馈: **"看起来很牛逼,但是有点抽象。我们直接构思不同的故事吧,太抽象了大家可能会看不懂。"**

→ **产品哲学层面的 pivot**:从"先定母题世界观再孵化作品"改为"**先做几个具体故事,母题从故事共性中自然浮现**"。母题决策暂缓 · 未寄之信隐喻自然生长进油画老师(字条 + 画布背后字)。

**Round 2 · Tipsy 数据 + 男向蓝海**
Serac 提供 tipsy示例.xlsx(30 条头部,29 条有效)。数据显示: **男角色 76% / 用户 76% 是女性 / 中文市场空白 / 男向白月光 TOP 榜零对标**。

→ 定位:**两条腿走路,但男向优先(蓝海)**。同时 Serac 抛出"完美妻子做晚餐"作为 fantasy 示例。

**Round 3 · 故事收敛到油画老师**
- 男向 4 候选(完美妻子/白月光同桌/隐居爱人/邻居)+ 补充 3 个(毕业前/家教/便利店)
- Serac 反馈: 完美妻子"不够性感"· 隐居爱人海外不适 · 邻居"我喜欢年龄比我小很多的"
- **Serac 亲选转向女向师生恋**: "我在高中当过兼职画画老师,有个小女孩喜欢我"— 灵感 + 身体记忆打底
- 拍板:35 岁已婚教授 × 20 岁大一新生 · 3-5 人小班多设计合法 1:1 · 2026 现代 setting · Cinematic 沉浸

**Round 4 · 8 → 5 Beat 精简(demo 优先)**
Serac 反馈: **"作为 demo 展示 太多了也不好,冗长也不好"**。

- 砍 · Beat 6(传谣 + 走廊擦肩)
- 合 · 原 Beat 2/3(阳光 + 不诚实)→ 融入新 Beat 2 里 Lucian 一句台词回忆
- 合 · 原 Beat 7 后续 + Beat 8 → 新 Beat 5(时间跳跃)

→ **关键 insight**: 新 Beat 2 直接 callback Beat 1(用户刚玩过) 比原来 callback Beat 3(玩过 3-4 分钟前) 强度更高 · **精简反而让 callback 更爽**。Demo 时长从 25+ tap 减到 12 次 tap。

**Round 5 · 日常时空 = 聊天,不是信**
Serac 反馈: **"信还是有点慢的,能不能直接和角色聊天呢"**。改成短对话 + 时间戳 + 现代通讯感 · 匹配 v1.5 §1.5 下沉定位。

**Round 6 · Claire wireframes 吸收**
Serac 分享同事 Claire 的 UI wireframes。核心贡献:
- **6 类 Beat 视觉词汇**(SCENE MARKER / NARRATION / CHARACTER SPEECH / YOUR ACTION / YOUR SPEECH / INNER THOUGHT / CHOICE)
- **Suggested ↔ Free-write 切换** · 同一 send button 驱动
- **Character-first opening** 替代书封面
- **WHERE YOU STAND 状态标签**(不是进度百分比)
- 三个沉浸方向对比(Cinematic / Editorial / Scene cards)· 我们选 **Cinematic**

Serac 拍板砍掉 Claire 的进阶特性(Direct scene / Direct others / +who? 用户 as 其他角色说话)· "感觉没必要,暂时先不用考虑"。

**Round 7 · UI/IA 定案**
- 底部 5 tab: 首页 / 探索 / 创作 / 消息 / 我
- 消息 tab 三视图: All / Characters / Story(⭕ 角色行 vs ▭ 故事行 · 微信式排序)
- 进入作品 3 tab 上限(v1 只做主角色 · **副角色 v1 不做**)
- 通道开启 UI 事件规格化:全屏动画 → 手机震动 → 全屏遮罩 → [打开] → 首次私聊 → 消息 tab 角色行升级为**金边卡片,置顶,金光扫过**
- 抖音尺度定案(大老板拍板)· 允许剧情擦边过程,fade-to-black 在关键 moment · 不做付费穿透

**Round 8 · in-place 回填 v1.5 + 用户旅程图**
- 玩法 v1.5:补 §2.5 UI 事件规格 · 新增 §2.7 呈现规范(Cinematic + 6 类 Beat + Suggested/Free-write)· §5 v1.5 范围拆"产品/玩法"+"UI/呈现"· §6.2 公开必备从 10 项扩到 12 项
- 用户旅程图 v1.0:**砍 U6.5**(7 天试用+信道窄化 · v1.5 §4.1 已砍)· U6 通道开启 UI 事件化 · U7 改名"日常时空长存" · 描述改为积分+订阅

### 关键新共识 · 一句话汇总

| 主题 | 决定 |
|---|---|
| 母题 | 暂缓 · 从下往上让母题浮现 · 不预设 |
| 用户定位 | 男向蓝海优先 · 女向红海必做 · 服务下沉不追求 delayed gratification |
| 第一个 demo | 油画老师(35 岁已婚 × 20 岁大一 · 现代 · 5 beat · Cinematic) |
| 精简哲学 | Callback 到用户**刚玩过**的比 callback 到几分钟前的强 |
| 商业尺度 | 抖音尺度 · fade-to-black 是产品机制不是审查妥协 |
| Beat 呈现 | Claire 6 类 vocabulary · Suggested ↔ Free-write |
| 沉浸方向 | Cinematic(3 方案选定)· 反糖水粉色少女漫 |
| 副角色 | v1 不做 · 保护单主角色 fantasy 浓度 |
| 通道开启 | UI 事件是产品最高单点仪式 · 金边卡片是留存黑魔法 |
| U6.5 | 砍(7 天试用+信道窄化 → 积分+订阅统一) |
| AI 主动 | 不扣积分 + 沉默降级("AI 知趣退")|

### 学到 · 决策哲学

- **抽象 vs 具体**: 老板/同事对齐要具体故事 · 抽象母题会失败。母题该从多个成功故事的共性提炼,不该 top-down 定死。
- **精简 vs 完整**: demo 场景下,精简 > 完整。用户体验 12 次 tap 优于 25 次 tap · callback 距离越近越爽。
- **下沉 vs 高线**: 我们服务下沉,不追求 delayed gratification · 首屏必须劲爆 · 每 session 必须有付费入口。
- **Serac 亲身经验优先**: 油画老师的成立是因为 Serac 有身体记忆(高中当过画画兼职老师)· 纯 AI 生成 + 有 lived experience 调教 > 纯 AI 生。
- **抄好抄的**: Tipsy 的入口审核逻辑("审 3 beat 内 hook 质量,不审设定完善度")· 消费→创作低门槛转化 · Claire 的 6 类 beat vocabulary · 都直接抄。
- **别抄坏的**: Tipsy 角色 20 轮丢记忆 · 一句话就换戏(迈克尔案例) · 付费穿透 fade-to-black · 都反着做。
