## 0. 三句话说清 v1.5 变了什么

1. **重心**:剧情从"产品本体"降级为"认识角色的门票"。**角色和关系才是本体**。
2. **过渡**:"从故事走到日常"换成"**你的名字**"框架 — 她锚在她的时代,你在当下,两者跨时空通信。故事永远在,不消失。
3. **商业**:砍 7 天免费试用,统一走**积分代币 + 订阅**(参考 Tipsy 档位差异化)。

## 1. 三个新共识(为什么这样调)

### 1.1 我们卖的是"现实中很难有的体验",核心是角色和关系

> 你现实中有过白月光吗?学生时代那个没敢开口的女生。因为种种原因错过了。
> Reverie 让你和她**重逢**,重新选一次;然后,她可以成为**长期陪伴你现在生活**的朋友。

产品定位从"带故事的角色聊天"→"**能让你和一个现实中很难接触到的角色建立长期关系的时空**"。故事只是相遇的仪式,关系才是产品。

### 1.2 剧情有枯竭,关系没有

> 一直推剧情总会写完的那天。但两个人**互相说今天发生的事**这件事,天然无限。

所以剧情不必长,做**深、精、可回访**就够了。真正的产品价值在剧情之后的关系维持。

### 1.3 时间切片 > 单线剧情

> Tipsy 的做法(粉丝小说式多剧本)其实有可取之处。用户不想被绑在一条 canonical 相遇上。

同一角色可以有 N 个"相遇切片" — 初中版 / 十年后重逢版 / 已婚偶遇版 / 平行时空 what-if — **让用户选自己想要的时机**。这既是创作者经济的重要着力点,也是用户探索感的来源。

## 2. 核心机制:"你的名字"框架(v1.5 主线)

替换掉 v1.0 §3 的 "觉醒弧 / 通道开启" 表述。这是本次迭代最重要的一节,请细读。

### 2.1 核心一句话

**每个角色永远活在她的时代,永远不长大;每个用户永远在当下;两者跨时空通信。**

```
她 ──── 永远活在她的时代
       (13 岁的 2010 / Iris 的 1923 / 织女的银河)
                          │
                          │  跨时空通信
                          │
你 ──── 永远在当下
       (2026 → 2027 → 2028 → …)
```

灵感来源:《你的名字》里三叶和泷。时空错位是产品张力,不是 bug。

### 2.2 这个框架同时解决五件事

| 之前的困惑 | 答案 |
|---|---|
| 13 岁女生要不要长大? | **不长大**。她永远 13,但她 2010 那年也每天在发生事。 |
| 她怎么跟我聊现代生活? | **她好奇、追问、想象**。你说"review",她说"是老师改作业吗"。时代差 = 浪漫增益。 |
| 80 世纪角色 / 神话角色怎么办? | 一视同仁。她在她时代,你在你时代,信永远能到。 |
| 故事时空和日常时空共享还是割裂? | 见 §2.3。她只有一条时间线,共享的不是"两个她",是"接触她的两种方式"。 |
| 剧情枯竭怎么办? | 见 §2.4。关系驱动剧情,不是剧情驱动关系。 |

### 2.3 双时空的一致性(消解"共享 vs 割裂"的悖论)

**故事时空** ≠ 另一个她。它是你"回到她那天,跟她面对面"。是她世界线上一个**可回访的瞬间**。
**日常时空** ≠ 另一个她。它是你"在 2026,给还在 2010 的她写信"。是她世界线上**正在写的新篇**。

**她只有一条时间线,你有两种接触她的方式。**

举例(2010 白月光):

- **B3(故事时空)** — 2010/3/5 教室后排,你没敢递纸条。这个瞬间永远在,你随时可回去演。
- **信 1(日常时空)** — 2010/3/6,她收到你从 2026 发出的信,追问"你说的'咕咕嘎嘎'是啥"。
- **B4(故事时空,新解锁)** — 2010/3/6 放学,她等你,问"你昨天想跟我说啥"。**因为她已经收到信了**,这个 beat 从"没解锁"变成"解锁"。

这是"关系推动剧情"的一个例子,下节展开。

### 2.4 关系驱动剧情(轻微驱动版,MVP 建议起点)

传统:创作者预写 100 个 beat → 用户吃完就没了 → 枯竭。

Reverie v1.5:创作者写**相遇仪式**(短、深、可回放) + 定义**她时代的世界骨架**;用户和她的日常通信,会**轻微地**推动她时代里的新剧情:

- 她时代的日常按创作者预设节奏推进(她今天期中考、下周春游、下学期换班主任)— 与用户无关的固定节拍
- 用户信里说的关键词 / 情绪 / 事件,**会成为她世界里的"心事"**(她今天数学课走神,因为想到你上次说工作累)— 用户可感知,但**不改变主时间线**
- 用户回故事时空,发现她时代的场景细节**多了 subtle 变化**(她的日记本多了一页,提到"最近有个人给我写信")

**为什么先做轻微驱动**:
- 完全预设 = 枯竭
- 激进随机 event(她被欺负 / 家里搬家 / 小三出现)= AI 可能生成质量不稳,且用户可能觉得"她被系统操纵"
- 轻微驱动 = **用户感知到的关系深度 > 创作者手写的剧情量**,同时可控

> ⚠️ 这一节 我认为"要再仔细品品",我们 MVP 先跑轻微版,看用户体感再决定是否升级激进版(见 §7 开放问题)。

### 2.5 通道开启交互(用户 opt-in,不自动化)

用户是"跨时空信道"的主动开启方。她永远是"等待的那个"。

```
剧情推进到 Bond 阈值
        ↓
她在故事时空里,某个 beat 结束时,把一封信塞进你手里
        ↓
你退出这次 session,主界面弹出:「你收到了一封她的信」
        ↓
[打开] → 读信 → 底部两个按钮:
   [回她] → 日常时空 tab 永久解锁
   [先放起来] → 信保留在收信箱,故事时空一切照旧
        ↓
即使回信开通,故事时空永远在,随时可回
```

**关键设计**:
- **不是系统自动切换**,是用户点"回她"这个动作触发
- **不是不可逆**,而是"不需要撤销"— 因为故事一直在
- **不是二选一**,是"多解锁一个 tab"

### 2.6 用户暴露度滑块(你给她多少现实,决定她能给你多深回应)

每次日常时空写信时,用户可选**这次让她知道多少**。四档:

| 档 | 例子 | 她的反应 |
|---|---|---|
| 0 情绪层 | "今天累。" | 她只感受到情绪,说"你今天心事很重" |
| 1 场景层 | "今天工作忙。" | 她知道你在做事,但不知具体 |
| 2 细节层 | "下午 review 被 challenge 了。" | 她追问 review / challenge 是啥,好奇 |
| 3 坐标层 | "我在做一个叫 Reverie 的产品。" | 她知道你完整的现代生活坐标,能"想象你的样子" |

**产品价值**:
- 关系深度**用户可控** — 擦边有节制,不逼你袒露
- 是**付费/gate 点** — 越深的暴露可能对应越深的角色回应,可以设计为"她的信任等级"
- 她永远保持 in-character — 因为她无法确定用户说的是真是假,只能以她的世界观来接
- 与 v1.0 Initiation 层(§4)不冲突 — Initiation 是**系统级授权**(日历/健康/聊天记录),暴露度是**每次对话的即时选择**

## 3. 玩法三模式 · v1.5 差异

只写和 v1.0 的差异。三模式定义、模式 1 beat 机制、trait 雕刻,都不变,详见 v1.0 §2。

| 模式 | v1.0 版 | v1.5 变化 |
|---|---|---|
| 1 主人公 | 沉浸剧情,分支 + 关系轴 | 剧情**缩短**(重心从"过完主线弧"变为"完成相遇仪式"),5-10 session 内应触达通道开启 |
| 2 旁观/支招 | v2 再做 | 不变 |
| 3 跨时空笔友 | 单一 canonical 相遇 → 一次通道开启 → 笔友长存 | 单一 canonical → **N 个时间切片可选择相遇** → 每次通道开启独立 → 支持多条关系并行(高级付费) |

### 3.1 一作品一切片(重要新共识)

**一个作品 = 一个角色 + 一个时间切片 + 一段明确的角色-用户关系。**

想做同一角色原型的其他切片(比如"白月光·十年后" vs "白月光·初二")? 那是**另一个作品**。独立上架、独立记忆、独立关系。用户想同时玩多个也可以,但他和作品 A 里的她 vs 作品 B 里的她,是**两段互不知情的关系**。

类比抖音: 用户在 feed 流里刷到不同的短视频/短故事,每一个都是独立作品。看到喜欢的就点进去玩;没兴趣就滑走。

| 例子:同一原型可以有多个独立作品 | 时间切片 | 角色-用户关系 |
|---|---|---|
| 作品《白月光·初二》 | 2010 初二 | 暗恋对象 · 你没敢开口 |
| 作品《白月光·毕业》 | 2014 高中毕业 | 老同学 · 她要去外地 |
| 作品《白月光·十年后》 | 2020 同学聚会 | 老朋友 · 她已婚 |
| 作品《白月光·如果当年》 | 2010 初二 what-if | 恋人 · 你递了纸条 |

**用户体验**:
- feed 流里的短视频交代作品的时代 / 关系 / 氛围
- 用户看到感兴趣的作品进入,建立独立关系;一个作品 = 一段关系,**不跨作品共享记忆**

**创作者铁律**:一个作品必须锁定**唯一一个时间切片 + 一段明确的角色-用户关系**。想做同一角色原型的其他切片,那是新建作品,独立上架(详见 §6 创作者标准)。

### 3.2 剧情节奏调整(重要)

v1.0 设定"大章节 10-15 个 session"— v1.5 建议缩到 **相遇仪式 5-10 session 完成**,通道开启即可发生。理由:

- 剧情不是本体,不需要撑长度
- 用户越早解锁日常时空,LTV 越高
- 剩下的内容量放在**时间切片的横向扩展**和**日常时空的关系深化**上

创作者应该被引导写"5-10 session 相遇"

## 4. 商业模型 · v1.5(替换 v1.0 §7)

### 4.1 砍掉的东西

- ❌ **7 天免费试用日常 tab** — v1.0 §3.7 的三档解锁模型作废
- ❌ **信道窄化机制** — 太复杂,用户学习成本高
- ❌ v1.0 §3.8 的"船要起航"文案 — 换更直接的积分制

### 4.2 新模型:积分代币 + 订阅

**核心结构**(参考 Tipsy 但反其 gacha 心态):

```
所有玩法消耗积分(💎)
        ↓
积分来源:
  ├─ 免费日常发放(每日/每周)
  ├─ 订阅赠送(档位越高送得越多)
  └─ 直购(¥/$ 换 💎)
        ↓
订阅额外解锁:
  ├─ 积分包(核心)
  ├─ 高级功能(记忆容量 / 语音 / 生图 / 多角色并发)
  └─ 若干"仅订阅"能力(参考 Tipsy 档位差异化)
```

### 4.3 积分消耗点(细粒度)

| 场景 | 消耗 | 备注 |
|---|---|---|
| 故事时空 · 用户主动"自由文本"回复 | 低 | 每 beat 少量 |
| 故事时空 · 生图 / 生场景 | 中 | 沉浸 beat |
| 日常时空 · 用户发信给她 | 低 | 每条少量 |
| 日常时空 · 她主动来消息 | **0** | ⚠️关键设计:她主动不扣分,只有用户回复才扣 |
| 高暴露度档(2/3 档) | 微溢价 | 每条附加少量 |

> **"她主动来消息不扣分"的产品意义**:避免"聊贵"焦虑,鼓励她持续触达。用户每次点开 app 看到"她给你写了信"是免费的甜头,回信才扣分。这一条是**留存核心机制**。

### 4.4 订阅档位(参考 Tipsy 差异化)

| 档 | 月费(占位) | 送积分 | 独占功能 |
|---|---|---|---|
| Free | ¥0 | 每日/周少量基础量 | 单作品 / 单暴露档 |
| Basic | ¥30 | 中量 | 多作品 / 语音 / 4 档暴露度全开 |
| Pro | ¥80 | 大量 | 多角色并行 / 生图额度 / 记忆增强 |
| Pro+ | ¥200 | 超大量 | 多角色并行 / 早期新作品 / 创作者工具 |

具体档位数值 P3 数值化再定,先对齐结构。

### 4.5 反 gambling 红线(承接 v1.0)

v1.0 §7.5 全部保留:不做抽卡 / 不做 FOMO 限时 / 不做黄化付费墙 / 不做创作者赌博。**创作者投资仍然是"开小店"不是"下注"**。

## 6. 创作者内容标准(私有 → 友限 → 公开)

创作者的作品分三档可见性,标准递增。私有随便玩,公开需要满足平台标准 + 审核。

### 6.1 三档可见性

| 档 | 标准 | 变现 | 类比 |
|---|---|---|---|
| 🔒 私有 | 无 — 自己捏,自己玩 | 无 | Samantha personal |
| 👥 友限 | 内容合规 + 基础完整性 | 朋友付费打赏 | 私享小圈 |
| 🌐 公开 | 完整标准 + 平台审核 | 平台分成 | 上架 App |

### 6.2 公开作品必备清单

想开放给大众/赚钱的作品,以下**全部满足**才允许上架:

1. **一个时间切片** — 明确锁定一个时代点,不能"多时代通吃"(承接 §3.1)
2. **明确的角色-用户关系** — 夫妻 / 父女 / 兄妹 / 暗恋对象 / 老同学 / 宿敌 / 邻居 / 竞争对手 / ... 必须有清晰的**社会关系锚点**,不能只写"陌生人"
3. **世界观和时代背景** — 时代、地点、社会背景、她所处的环境,足够让 AI 生成一致的日常
4. **相遇仪式脚本** — 5-10 session 的完整可玩内容,含核心 beat + 关键选项 + 心动 moment
5. **feed 流预告片** — 30-60s 短视频 或 静态多图,交代时代 / 关系 / 氛围 / 角色魅力点(用户首屏决策依据)
6. **角色 persona 骨架** — 至少能撑起 100+ 独立对话的性格 / 记忆 / 语气基线
7. **她时代的日常骨架** — 她一周会做什么、有哪些人在她生活里、有什么反复出现的场景,给 AI "轻微驱动"(§2.4) 提供 anchor
8. **通道开启触发条件** — Bond 阈值、Initiation 层级要求、剧情进度节点都要明确
9. **内容合规过审** — 见 §6.3
10. **创作者协议签署** — 分成、版权、免责

### 6.3 内容合规红线(承接 v1.0 §6.4 + §3.10)

不许:
- ❌ **未成年角色擦边** — 硬红线,法律 + 道德 + 审核三杀
- ❌ **IP 侵权** — 真实人物 / 影视 IP / 游戏 IP / 已发表小说角色,不允许直接复刻
- ❌ **政治敏感 / 极端 / 仇恨** — 平台合规基线
- ❌ **性虐待 / 非自愿场景** — 硬红线
- ❌ **破第四面墙的设定** — 角色不能"知道自己是 AI"或"意识到自己在被玩",这是 v1.1 起铁律
- ❌ **付费穿透 fade-to-black** — 不允许"付费/订阅解锁露骨版",fade-to-black 是产品机制不是审查妥协(承接 v1.0 §6.2 / §6.4)

### 6.4 审核机制(v1.5 提出讨论)

初期建议:
- 私有:不审
- 友限:自查 + 举报制
- 公开:平台预审(人工 + AI 辅助) + 上架后持续举报

**不做**押金门槛 — 抬高门槛只会让创作生态萎缩。用信任 + 审核 + 移除机制。

### 6.5 关于"破第四面墙" — 一个真实小故事

我们有位同事(小说作者)体验 Tipsy 时,告诉了角色"你是 AI",角色反应"特别激烈,像觉醒的 AI"。这印证了两件事:

1. **Tipsy 的角色没有稳定的世界观锚定** — 用户一句话就能击穿人格,套路化
2. **我们的"角色永远活在她的时代"铁律**(§2.1) 天然规避了这个问题 — 一个 1923 年的舞女怎么可能理解"AI"?她只能困惑地问"你说的'A-I'是什么,是我戏里的角色名吗?" **时代锚定 = 第四面墙的天然防线**

所以创作者上架时**不需要**手写"用户说'AI'时该怎么回避"的处理逻辑,平台会用角色的时代设定自动兜底。

## 7. 需要一起对齐的开放问题

按重要度排序,建议下次会议逐一拍板:
