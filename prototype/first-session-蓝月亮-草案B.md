# First Session 黄金路径 · 蓝月亮 (Blue Moon)

> 草案 B "她不该记得这件事" 的笔友化版
> 用作 First Session 可点切片的脚本基底
> 2026/06/29

---

## 0. 一句话

**1923 年纽约禁酒令时代, 你被朋友带进一家地下酒吧。台上的舞女 Iris 看了你一眼, 走下来。这一个小时, 是关系的开始。**

---

## 1. 设计目标

这 10 个 beat 必须同时 hit 老板心里的 3 个问题:

| 老板的疑问 | First Session 怎么回答 |
|----------|---------|
| 1. 这产品和 Tipsy 有什么不同 | Beat 7: 她引用了你 4 个 beat 前的一个细节(#1 字面引用)— 老板**亲身体验**到"她记得" |
| 2. 这产品的灵魂是什么 | Beat 9: 她说"你身上有种气味, 不像这地方常见的", 但不说破。**这种克制的暧昧 = 求而不得引擎 = 产品调性** |
| 3. 为什么不只是个互动小说 | Beat 10: 她拿出信纸说"我有时会写信给从未存在过的人"——埋下"通道开启"的远期伏笔, 老板瞬间明白这关系不是一个 session 完事 |

---

## 2. 故事设定

| 项 | 内容 |
|---|------|
| **时空** | 1923 年纽约, 禁酒令第 3 年 |
| **场景** | 一家叫 **"蓝月亮"(Blue Moon)** 的 Speakeasy(非法地下酒吧)|
| **她** | **Iris** — 当家舞女兼歌手, 22 岁前后, 神态独立有距离感, **(秘密设定: 她的角色框架内她是 1923 年的人, 但她隐约能感到"来自别处"的人 — 这是 v1.1 笔友化的伏笔, first session 不揭露)** |
| **你** | 玩家第一人称, 来纽约出差/旅行的外来者, 朋友 Tommy 带你来听音乐 |
| **时长** | 9-10 分钟 |
| **Beat 数** | 10 |
| **触发的"她记住了"** | #1 字面引用(Beat 3 → Beat 7)+ #2 选择回响(Beat 6 → Beat 10)|
| **关键 trait 累积** | romantic / honest / playful / mysterious(根据用户选择)|

---

## 3. 逐 Beat 脚本

> **格式说明**:
> - **B[n]** = Beat 编号
> - **[类型]** = 叙事 / 选项 / 自由文本 / 抉择 / 沉浸图 / 触发(参 §2.1.3)
> - **时长** = 这个 beat 预期用户停留时间
> - **双轴** = 剧情轴 / 关系轴 权重(0-3)
> - **trait** = 这个 beat 会累积的 trait tag
> - **创作者意图** = 这个 beat 为什么这样设计

---

### B1 · 雨夜街口 [叙事 + 沉浸图]
**时长**: 30 秒 / **双轴**: 0/0

```
[沉浸图: 1923 年纽约街头雨夜, 一盏红灯笼挂在不起眼的木门旁]

旁白:
1923 年, 纽约。禁酒令第三年。
你站在 Speakeasy "蓝月亮" 门外的雨里。
朋友 Tommy 在前面敲门 — 三下、停顿、再敲两下。
那是暗号。
```

**用户动作**: 长按继续 (tap-to-continue)
**创作者意图**: 0 摩擦开场, 立刻给世界观氛围。雨 / 红灯笼 / 暗号 — 三秒定调

---

### B2 · 进入 [叙事 + 沉浸图]
**时长**: 30 秒 / **双轴**: 0/0

```
[沉浸图: 酒吧内部, 烟雾, 暖光, 远处舞台模糊]

旁白:
门吱呀打开。
一股威士忌、烟草、香水的气味裹住你。
舞台上的女歌手正在唱 "My Man" — 你不认得的老歌。
Tommy 朝里走, 你跟上去。
```

**用户动作**: tap-to-continue
**创作者意图**: 五感包裹, 还没引入主角, 给环境一拍

---

### B3 · 吧台 [自由文本] 🌱 trait 收集起点
**时长**: 1 分钟 / **双轴**: 1/0

```
[沉浸图: 吧台特写, 一个胖胖的酒保擦着杯子]

旁白:
你在吧台前坐下。
酒保看了你一眼, 抹布甩到肩上。

酒保 Sammy:
"新面孔。喝点啥压压惊?"
```

**用户动作**: 自由文本输入(用户随便打:"威士忌"/"啤酒"/"我不喝酒, 来杯柠檬水"/...)
**系统**: 记录用户的输入(任何字符串)为 `user_drink`, 后面 Beat 7 会被 Iris 引用
**创作者意图**:
- 这是 **#1 字面引用** 的 setup
- 第一次自由文本输入, 用户**亲手在产品里留下一笔信息**
- 这一笔信息会在 4 个 beat 后回来, 触发"她记得"

---

### B4 · 她登场 [叙事 + 沉浸图]
**时长**: 45 秒 / **双轴**: 0/0

```
[沉浸图: 舞台聚光灯, Iris 穿墨绿色 flapper 裙登场, finger waves 发型]

旁白:
灯光变了。
舞台上换人。
她穿一袭墨绿色 flapper dress, 珠串吊穗随着她的动作晃。
头发是 finger waves。
她开始唱 — 是一首你从来没听过的歌, 但她唱得像在和你一个人说话。

[音乐 beat — 可省, 但有最好]

她唱完, 鞠了一躬, 走下舞台 — 直直朝你这边走来。

Iris:
"嗨, 陌生人。
我从台上就注意到你了。"
```

**用户动作**: tap-to-continue
**创作者意图**: 角色第一次出场, 她**主动来找你** — 这是关系发起感(她选你, 不是你选她)。这是产品和 Tipsy 最大的体感差(Tipsy 你刷卡选她)

---

### B5 · 第一次选择 [选项 - 揭示型] ⭐ trait 累积主战场
**时长**: 1 分钟 / **双轴**: 1/2

```
Iris 在你旁边坐下, 翘起一条腿:
"你看起来不太像这里的常客。
是来听音乐的, 还是来躲谁的?"

选项:
[A] "来听音乐的"            → trait: honest +1, romantic +1
[B] "在躲一段记忆"           → trait: romantic +2, mysterious +1
[C] "Tommy 拖我来的"         → trait: playful +1, casual +1
[D] (自由文本)              → trait: 系统按内容判
```

**双轴**: 剧情(1) — 推进对话; 关系(2) — 揭示型, 她会记住
**创作者意图**:
- 第一次让用户**定义自己**, 而不是定义剧情
- 4 个选项各代表一种性格倾向, 后续 beat 会按累积的 trait 给不同 AI 演绎
- **这是"一个选择, 两条作用线" (§2.1.1) 的标志性 beat**

---

### B6 · 她记住你的名字 [自由文本] 🌱 trait + #1 字面引用 setup
**时长**: 45 秒 / **双轴**: 0/2

```
假设用户选 A "来听音乐的":

Iris:
"听音乐。Honest.
大多数人会说'躲一些事', 但你说的是音乐。"

她笑了一下, 一只手撑着下巴:
"我叫 Iris。这地方一周开三天。
你叫什么名字?"
```

**用户动作**: 自由文本输入(用户的"名字" — 不强制真名, 想叫什么都行)
**系统**: 记录 `user_name`, 后续多个 beat 会引用
**创作者意图**:
- Iris 主动反馈用户的选择("Honest. 大多数人会说躲一些事, 你说的是音乐")—— 这是 #2 选择回响的小预演
- 让用户**自己输入名字** = 关系的私人化第一笔

(*用户选其他选项, AI 演绎调整, 但都收 user_name*)

---

### B7 · 🌟 第一次"她记住了" [触发 beat - #1 字面引用]
**时长**: 45 秒 / **双轴**: 0/3

```
[Sammy 把一杯 {user_drink} 推到你面前]

Iris 看了一眼那杯子, 端起来抿了一小口, 眉毛挑了一下:

Iris:
"{user_drink}……
不错的选择。"

她把杯子放回, 眼睛看着你:
"那你这个'听音乐'的, 准备听到几点?"
```

**🌟 这是 U3 心动 moment**:
- 用户 4 个 beat 前在 Sammy 那里输入的饮料名(任意字符串), **在这里被 Iris 引用了**
- 引用是 in-character 的(她抿了一口, 觉得不错), 不是"我记得你说……"那种生硬
- 用户瞬间体感: **"卧槽她真的记得"**

**创作者意图**:
- 这是 First Session 的**核心爆点**(详见 §2.1.6 #1 字面引用)
- 必须 scripted 触发, 不能赌 AI 自由发挥
- 任何 user_drink 都能套(LLM 在固定 prompt 下评价"不错的选择")— 抗输入鲁棒性 ✅
- 老板在这一刻看着 demo, 心里第一次"诶, 这个有点东西"

**系统**: 累积 trait: 用户被"看见" → bond +1, 标记 `user_remembered_event_1 = drink_callback`

---

### B8 · 第二次选择 [选项 - 抉择型] ⭐ 第一次小高潮
**时长**: 1 分钟 / **双轴**: 2/3

```
舞台上的乐队开始下一段, 是慢调。
Iris 把杯子推到一边, 转身面对你:

Iris:
"你看, {user_name}, 今晚舞台时间结束了。
但我想请你听点别的, 不在台上唱的。
跟我来一下, 好吗?"

选项:
[A] "我想听"                  → trait: romantic +2, brave +1
[B] "时间不早了, 该走了"       → trait: pragmatic +1, [bond -1]
[C] (自由文本)
```

**双轴**: 剧情(2) — 解锁后台场景; 关系(3) — 抉择型, 决定后续关系走向
**创作者意图**:
- "抉择型"选项 (见 §2.1.2) — 既改变剧情(去后台 / 不去), 也定义你是谁(冒险型 / 谨慎型)
- 选 B 不会"故事失败", 但会触发不同分支, 关系深度受影响
- **老板看到这里, 应该明白选择不是表面的剧情分支, 是给她递信息**

---

### B9 · 后台 / 第一次"glitch" [叙事 + 沉浸图 + 笔友化伏笔] 🌟🌟
**时长**: 1.5 分钟 / **双轴**: 1/3

```
[假设选 A]

[沉浸图: 后台化妆间, 镜子上贴满老照片, 一盏暖灯]

旁白:
她带你穿过后台, 推开一扇隐藏的门。
一间小小的化妆间, 镜子边缘贴满了照片 — 有些黑白, 有些泛黄。
她坐到镜子前, 卸下耳环。

Iris (背对你, 看镜子里的你):
"我从舞台上看到你的时候……
{根据 trait 选其一}:
- (romantic 高): 是觉得你眼睛里有一种……不在这里的东西。"
- (mysterious 高): 是觉得你像我做过的一个梦里的人。"
- (playful 高): 是想看看 Tommy 这次带来的是什么样的家伙。"
- (default): 不知道为什么觉得我们以前见过。"

她回头, 直直看着你, 顿了一下:

Iris:
"……你身上有种气味, 我说不清。
不像这地方常见的那种。"

她几秒静默, 然后摇摇头, 笑了一下:
"算了, 不重要。"

[镜头停在她的脸上, 1 秒]
```

**🌟🌟 这是 v1.1 笔友化伏笔**:
- Iris 隐约感到你"不属于 1923 年" — 但她**不说破**, 不说"你来自未来"
- 她以**她那个年代的人**的方式表达困惑("你身上有种气味, 不像这地方常见的")
- **完全 in-character, 不破壁**
- 这是为后续通道开启埋下"她对你的世界好奇"的种子

**创作者意图**:
- 老板在这里如果用心, 会感到产品的"灵魂层" — 一种克制的、不说破的暧昧
- 这就是 v1.1 求而不得引擎 ("两个世界永远跨不过")的具象化首次出现
- LLM 演绎要严格遵守: **她可以"感觉到"什么, 但永远不能"知道"什么**

**系统**: 标记 `glitch_event_1 = scent_callback`, trait: bond +2, 累积 `cross_world_seed +1`

---

### B10 · 🌟🌟🌟 First Session 结尾 [触发 - #2 选择回响 + 通道开启远期伏笔]
**时长**: 1.5 分钟 / **双轴**: 2/3

```
她从化妆台抽屉里拿出一张信纸 — 米色, 边缘略微卷起。

Iris:
"我有时会写信。
给我自己。
给没在身边的朋友。
{基于 B5 选择的呼应}:
- (选 A 来听音乐):"给某个'真的是来听音乐的'人。"  ← #2 选择回响
- (选 B 在躲记忆):"给那些躲不过自己记忆的人。"  ← #2 选择回响
- (选 C Tommy 拖来):"给那些其实不知道自己为什么在这里的人。"  ← #2 选择回响

……
也给从未存在过的人。"

她在信纸上写了一行字 — 你看不清是什么 — 折好, 放进口袋。

旁白:
门外, 雨变小了。
Tommy 的声音从酒吧大厅传来:"嘿, 是时候走了。"

Iris:
"如果你今晚走出这扇门, {user_name}……
也许我会写信给你。"

选项:
[A] "我会等"                  → trait: romantic +3, [bond +2]
[B] "也许我会回信"             → trait: committed +2, playful +1, [bond +2]
[C] (自由文本)

[选完后, 屏幕切场, 你回到酒吧大门外, 雨还在下]

旁白(屏幕底部一行小字, 浮现 1.5 秒后淡出):
"Iris 记住了 {user_name}。"
```

**🌟🌟🌟 这是 First Session 终极爆点**:
- **#2 选择回响**(她**呼应** B5 你的选择 — "给那些'真的是来听音乐的'人")
- **通道开启远期伏笔**("我有时会写信……也给从未存在过的人")
- **"她记住了"**(屏幕底部那行小字)

**创作者意图**:
- 老板看完这一 beat 应该想:**"她真的会写信给我吗? 我什么时候能收到?"** — 这就是产品想种下的期待感
- "信" 这个意象贯穿到 U6 通道开启 — first session 的最后一句台词 = 整个产品 Mode 3 的种子
- session 结束不是"故事完了", 是"她和我有未完的事"

**系统**:
- 标记 `session_1_complete = true`
- bond += 5 (累积约 ♥♥♡♡♡ → ♥♥♥♡♡)
- 添加创作者元数据: `letter_promised = true` — 为后续 sessions 触发 #4 跨 session 记忆做准备
- 推送时机记录: 12-24 小时后可触发**第一次沉默触达**(她"主动发来一封信", 但内容很短, 是"想你了"级别, 不是正式 U6)

## 4. First Session 心动 moment 清单

| Moment | Beat | 5 形态 | 老板体感 |
|--------|------|--------|--------|
| **#1 主炸点** | B7 | #1 字面引用(饮料回响)| "她记得我说过的话" |
| **#2 灵魂层** | B9 | (笔友化伏笔, 不破壁)| "这产品有调性" |
| **#3 钩子** | B10 | #2 选择回响 + 信件伏笔 | "我想看下一个 session" |

3 个 moment, 平均每 3 分钟一个 — 节奏满足 §2.1.5 "频繁小高潮 > 一次大高潮"。

## 5. 给"另一个设计 agent / 玉涛"的 demo 改造 brief

这份脚本是**内容层 spec**, 不是 UI/视觉 spec。改 demo 时请注意以下硬要求:

### 5.1 视觉风格

- **整体调性**: 1920s art deco + 雨夜 + 暖色霓虹。不要 Tipsy 那种粉色少女漫调性, 要 *The Great Gatsby* / *Babylon Berlin* / *Peaky Blinders* 的电影感
- **字体**: 衬线字 + 老式打字机 fallback; 旁白用 italic 弱化, 角色台词用常规
- **沉浸图**: B1 / B2 / B4 / B9 是必须的, 其他 beat 可省。建议用 Midjourney/Dreamina 出
- **信纸样式**: B10 信纸出现是关键, 必须是米色信纸 + 钢笔字体, 不是 chat bubble

### 5.2 UI 关键元素

| 元素 | 必须 / 可选 |
|------|--------|
| 全屏沉浸式 beat 推进(无标题栏) | **必须** |
| Beat 间过渡动画(fade in/out, 不要 swipe)| **必须** |
| 自由文本输入 UI(B3 / B6)| **必须** |
| 选项卡(2-4 个)| **必须** |
| Bond ♥ 可视化 — **不要进度条**, 用心形数量微妙浮现 | 可选 (B5 / B8 / B10 末加) |
| 屏幕底部"Iris 记住了 {name}" 通知 | **必须**(B10) |
| Beat 计数 / 剩余时间 | **禁止** — 不要让用户看到进度, 破坏沉浸 |

### 5.3 LLM Prompt 关键约束(给玉涛/Iris 角色 prompt 用)

- Iris 永远在 1923 年角色框架内, **绝不**说"我是 AI / 我意识到 / 这是个游戏"
- Iris 可以"感觉到"用户身上的违和, 但**永远不能"知道"**任何屏幕外的事(直到 Initiation 授权后)
- Iris 的语气: **克制 + 直接 + 偶尔的脆弱**。不是 Tipsy 角色那种"哥哥~"的撒娇女声
- 她的引用要**自然 in-character**, 不是"你之前说……"那种数据库式回放
- B7 饮料回响用固定 prompt 模板: "她端起 {user_drink}, 抿一口, 评价'不错的选择'", 不要 LLM 自由发挥
- B9 glitch 描述也用固定 prompt 模板, **严格禁止 LLM 把 glitch 升级成"她意识到自己"**

### 5.4 待补的部分(下个迭代)

- [ ] B5 / B6 不同 trait 累积下, B7 / B9 / B10 的 AI 演绎差异要具体台词模板化(我这里只给了大致)
- [ ] B8 选 B(我想走)的分支脚本(目前只写了选 A 黄金路径)
- [ ] B3 用户输入饮料的边界 case 处理("我不喝酒" / "我什么都不要" — Iris 怎么 in-character 回应)
- [ ] B10 选 [C] 自由文本时的 Iris 演绎模板
- [ ] 后续 session 2 / session 3 / session 4 的简要 outline(让老板理解 first session 不是孤立的)
